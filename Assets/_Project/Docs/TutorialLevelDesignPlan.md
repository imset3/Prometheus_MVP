# 튜토리얼 레벨디자인 확정안

## 1. 범위와 우선순위

- 대상은 `Assets/Scenes/TutorialScene.unity` 하나다.
- 챕터 1의 내용은 포함하지 않으며, 마지막에는 Chapter 1 전환 버튼만 유지한다.
- 현재 대화에서 확정한 방향이 HTML 프로토타입과 노션보다 우선한다.
- HTML 프로토타입에서는 HQ 인트로, 테우스 추적, 독립된 훈련장, 단계별 훈련, 크리온 장비 전달의 흐름을 차용한다.
- 추가 확정 사항은 구역 간 페이드 전환, 보스 전 일반 전투 2개 구역, 크리온의 더블점프 부츠와 추가 모듈 동시 전달이다.
- 최종 아트 연결 전까지 모든 공간은 단색 도형과 기본 콜라이더로 제작한다.

## 2. 전체 플레이 흐름

```text
Z01 아다마스 HQ
  인트로 → 테우스 소개 → 테우스 추적
        │ T01
        ▼
Z02 훈련장
  하강 진입 → 대시 → 점프/활공 → 기본 공격 → 펄스
        │ T02
        ▼
Z03 크리온 장비 전달 구역
  크리온 등장 → 더블점프 부츠 + 보조 모듈 → 장비 시험 → 릴레이
        │ T03
        ▼
Z04 외곽 전투 I
  기본 적 조합 전멸
        │ T04
        ▼
Z05 외곽 전투 II
  강화 적 조합 전멸 + 새 장비 활용
        │ T05
        ▼
Z06 광물 저장고
  보스 진입 대화 → 헬테 소개 → 보스전 → 결과 화면
```

## 3. 월드 좌표 계획

메인 카메라는 Orthographic Size `5`, 16:9 기준으로 세로 약 `10`, 가로 약 `17.8`유닛을 보여준다. 구역 사이에 최소 약 `80`유닛의 공백을 두어 다른 구역이 카메라에 노출되지 않도록 한다.

각 구역의 자식 오브젝트는 가능한 한 로컬 좌표로 배치한다. 아래 Origin은 구역 Root의 월드 좌표다.

| 구역 | Root 월드 Origin | 로컬 X 범위 | 주요 로컬 Y | 플레이어 시작 | 출구 | 카메라 Center X 범위 | Fixed Y |
| --- | --- | --- | --- | --- | --- | --- | --- |
| Z01 HQ | `(0, 0, 0)` | `-40 ~ 40` | 바닥 `0`, 시작 단상 `1` | `(-30, 1)` | `(34, 0)` | `-31.1 ~ 31.1` | `1` |
| Z02 훈련장 | `(200, 0, 0)` | `-65 ~ 65` | 진입 `2`, 훈련 바닥 `-4` | `(-55, 2)` | `(55, -4)` | `143.9 ~ 256.1` | `-1` |
| Z03 장비 전달 | `(400, 0, 0)` | `-30 ~ 30` | 바닥 `0`, 시험 단차 `2.5/4` | `(-22, 0)` | `(24, 0)` | `378.9 ~ 421.1` | `1` |
| Z04 외곽 전투 I | `(600, 0, 0)` | `-45 ~ 45` | 바닥 `0`, 보조 단차 `2` | `(-37, 0)` | `(37, 0)` | `563.9 ~ 636.1` | `1` |
| Z05 외곽 전투 II | `(800, 0, 0)` | `-50 ~ 50` | 바닥 `0`, 전투 단차 `2/4` | `(-42, 0)` | `(42, 2)` | `758.9 ~ 841.1` | `2` |
| Z06 저장고 | `(1000, 0, 0)` | `-60 ~ 60` | 진입 `0`, 보스 바닥 `0` | `(-52, 0)` | 결과 화면 | `948.9 ~ 1051.1` | `1` |

카메라 수치는 그레이박스 플레이 테스트 후 최대 `±2`유닛 범위에서 조정할 수 있다. 구역 Origin과 순서는 변경하지 않는다.

## 4. 목표 씬 계층

기존 `StageSystems`, `PlayerRoot`, `TutorialGuideCompanion`, `TutorialObjectiveBeacon`, `TutorialAudioRoot`, `TutorialHUD`는 공용 시스템으로 유지한다.

```text
StageRoot
├─ StageSystems
├─ PlayerRoot
├─ TutorialGuideCompanion
├─ TutorialObjectiveBeacon
├─ TutorialAudioRoot
└─ TutorialLevelRoot
   ├─ Z01_HQ
   │  ├─ GeometryRoot
   │  ├─ NarrativeRoot
   │  ├─ GameplayRoot
   │  └─ Anchors
   ├─ Z02_Training
   ├─ Z03_CryonReward
   ├─ Z04_ExteriorCombatA
   ├─ Z05_ExteriorCombatB
   └─ Z06_OreStorageBoss
```

각 구역의 `Anchors`에는 다음 Transform을 사전 배치한다.

- `EntrySpawn`
- `ExitTrigger`
- `GuideAnchor`
- `ObjectiveAnchor`
- `CameraBounds`
- 전투 구역에는 `EncounterSpawnRoot`와 `EncounterExitGate`

런타임에는 새 GameObject를 생성하지 않고 사전 배치된 Root와 오브젝트의 활성 상태만 변경한다.

## 5. 구역별 설계

### Z01 아다마스 HQ

- 왼쪽 시작 단상에 프로메, 에온, 아르온, 엘륨을 배치한다.
- 시작 대화가 끝나면 테우스 소개 카드를 표시한다.
- 소개 종료 후 테우스가 최소 3개의 웨이포인트를 따라 오른쪽 출구로 이동한다.
- 플레이어가 현재 테우스 웨이포인트 반경 안에 도착해야 다음 웨이포인트가 열린다.
- 마지막 웨이포인트에서 훈련장 이동 대화 후 T01을 실행한다.

### Z02 훈련장

- 진입점은 훈련 바닥보다 약 `6`유닛 높게 두어 HTML의 하강 진입 인상을 유지한다.
- 사다리 기능은 현재 구현되어 있지 않으므로 2단계 그레이박스에서는 경사로와 짧은 낙하 통로를 사용한다.
- 실제 사다리가 필요하다고 확정될 때 별도 등반 기능 작업으로 교체한다.
- 훈련 공간은 좌우 벽으로 닫힌 하나의 경기장으로 구성한다.
- 훈련 순서는 `대시 회피 → 점프/활공 회피 → 기본 공격 → 펄스 공격`으로 고정한다.
- 모듈 트리 교육은 장비 전달 구역으로 이동한다.

### Z03 크리온 장비 전달 구역

- 훈련 종료 직후 크리온이 등장하고 대화와 소개 카드를 표시한다.
- 하나의 장비 전달 연출에서 더블점프 부츠와 추가 보조 모듈을 함께 지급한다.
- 보상 직후 모듈 트리를 열어 새 모듈을 확인하도록 한다.
- 낮은 단차, 더블점프 필수 단차, 모듈 사용 지점을 순서대로 배치한다.
- 시험 구간 종료 후 릴레이를 활성화하면 T03이 열린다.
- 추가 모듈의 명칭과 효과는 5단계 구현 전에 확정하며, 현재는 보상 슬롯과 시험 지점만 예약한다.

### Z04 외곽 전투 I

- 첫 실전 전투로 사용한다.
- 기본 근접 적 중심의 읽기 쉬운 조합으로 구성한다.
- 해당 구역의 등록된 적이 모두 처치되기 전까지 출구 게이트를 닫는다.
- 전멸 시 짧은 테우스 대화 후 T04를 실행한다.

### Z05 외곽 전투 II

- 보스 직전 난이도 상승 구역이다.
- 높이 `2`와 `4`유닛의 단차를 배치해 더블점프의 실전 사용을 유도한다.
- 추가 모듈을 사용하면 유리한 적 배치 또는 장치를 포함한다.
- 전멸 전에는 출구를 닫고, 전멸 후 저장고 진입 대화와 T05를 실행한다.

### Z06 광물 저장고

- 입구 복도와 넓은 보스 경기장을 분리한다.
- 입구에서 대화와 헬테 소개 카드를 표시한 후 경기장 문을 닫는다.
- 헬테 처치 시 기존 튜토리얼 완료 이벤트와 결과 UI를 사용한다.
- Chapter 1은 결과 화면의 전환 버튼까지만 범위에 포함한다.

## 6. 전환 규칙

모든 구역 전환은 하나의 공통 순서를 사용한다.

1. 출구 또는 클리어 조건을 잠근다.
2. 플레이어 입력을 비활성화한다.
3. 대화가 열려 있다면 종료를 기다린다.
4. 화면을 페이드 아웃한다.
5. 이전 구역 Root를 비활성화한다.
6. 다음 구역 Root를 활성화한다.
7. 플레이어, 테우스, 비콘을 다음 Anchor로 이동한다.
8. 카메라 Center X 범위와 Fixed Y를 갱신하고 즉시 스냅한다.
9. 체크포인트와 현재 구역 ID를 저장한다.
10. 페이드 인 후 다음 목표와 대화를 시작한다.
11. 플레이어 입력을 다시 활성화한다.

페이드 시간의 초기값은 Out `0.35초`, 완전 암전 유지 `0.15초`, In `0.45초`로 정한다.

## 9. 1단계 완료 기준

- 단일 씬과 페이드 전환 방식을 유지한다.
- 6개 구역의 순서, Origin, 크기, 시작점, 출구, 카메라 범위가 확정되었다.
- 보스 전 일반 전투 구역 2개가 별도 단계로 정의되었다.
- 크리온이 부츠와 추가 모듈을 함께 전달하는 위치와 후속 시험 구간이 확정되었다.
- HTML에서 차용할 요소와 현재 방향으로 확장할 요소가 구분되었다.
- 2단계에서는 이 문서를 기준으로 도형 그레이박스와 목표 계층만 재배치한다.

## 10. 2단계 구현 상태

`TutorialScene`의 `StageRoot/TutorialLevelRoot` 아래에 다음 6개 구역 그레이박스를 배치했다.

- `Z01_HQ_Prologue` — 시작 단상, 본부 이동 바닥, 출구 게이트
- `Z02_TrainingRoom` — 하강 계단, 긴 훈련 레인, 훈련 구간 표식
- `Z03_CryonReward` — 장비 지급 단상, 더블 점프 시험 발판, 릴레이 단상
- `Z04_ExteriorCombat_A` — 첫 전투 경기장, 엄폐물, 출구 게이트
- `Z05_ExteriorCombat_B` — 높이 차가 있는 두 번째 전투 경기장, 출구 게이트
- `Z06_OreStorage_Boss` — 저장고 입구, 넓은 보스 경기장, 보스 위치 표식

각 구역에는 `GeometryRoot`, `NarrativeRoot`, `GameplayRoot`, `Anchors`를 공통으로 두었다. `Anchors`에는 구역 성격에 따라 `EntrySpawn`, `ExitTrigger`, `GuideAnchor`, `ObjectiveAnchor`, `CameraBounds`, `EncounterSpawnRoot`, `EncounterExitGate`를 배치했다.

현재는 `Z01_HQ_Prologue`만 활성화하고 나머지는 비활성화했다. 기존 기능 오브젝트는 삭제하지 않고 이전 레이아웃 Root만 비활성화해 후속 단계에서 새 Anchor에 재연결할 수 있게 보존했다. 플레이어와 튜토리얼 시작점은 HQ 시작 단상으로 이동했고 초기 카메라 범위도 HQ에 맞췄다.

검증 결과:

- Tutorial Scene Validator 통과
- EditMode 테스트 21개 전체 통과
- Unity 콘솔 오류 및 경고 없음

다음 단계는 공통 페이드 전환 컨트롤러와 구역 활성화/플레이어 이동/카메라 범위 갱신을 연결하는 작업이다. 이후 기존 대화·퀘스트·적 스폰 기능을 각 구역의 Narrative/Gameplay Anchor에 순서대로 연결한다.

## 11. HQ 그레이박스 2차 보정

`Z01_HQ_Prologue`를 HTML의 시작 장면과 실제 플레이 카메라를 기준으로 다음 3개 공간으로 재구성했다.

1. **브리핑 룸** — 에온, 아르온, 엘륨, 프로메, 테우스가 한 화면에 모이는 인트로 공간
2. **메인 홀** — 테우스를 따라 기본 이동을 익히는 짧은 연결 구간
3. **출구 전실** — 마지막 안내 대화와 훈련장 전환을 준비하는 공간

NPC는 `PlaceholderActors_ART_SLOTS` 아래의 `ART_SLOT_Eon`, `ART_SLOT_Aron`, `ART_SLOT_Elium` 도형으로 배치했다. 이 도형들은 기능 참조를 갖지 않으며 추후 실제 모델로 교체하는 명시적인 자리 표시자다.

`IntroDialogueAnchors`, `TheusGuidePath`, `CameraShot_Intro/Hall/Exit`, `HQ_Checkpoint_Intro/Exit`를 추가했다. 기존 `AdamasHeadquartersRoot`의 시작점과 동행 경로 Transform은 삭제하지 않고 새 HQ 위치에 맞춰 이동해 기존 참조를 보존했다.

플레이 모드 첫 화면에서 기존 테우스 소개 UI와 새 브리핑 룸이 함께 정상 출력되는 것을 확인했다. HQ 카메라 범위는 실제 크기를 가진 Trigger Anchor로 보정했으며, Validator와 EditMode 테스트 21개가 모두 통과했다.

## 12. HQ 기능 연결 및 수직 구조 검증

HQ 첫 퀘스트의 완료 조건을 단순 `MovementPerformed`에서 `TUTORIAL-HQ-EXIT`의 `PortalUsed` 신호로 변경했다. 이제 이동키를 한 번 누르는 것으로 훈련장 단계가 조기에 시작되지 않으며, 실제 HQ 출구에 도착해 페이드 전환이 시작될 때 다음 퀘스트로 넘어간다.

인트로 대화가 끝나면 `TutorialGuideRouteHost`가 `TheusGuidePath`의 5개 지점을 순서대로 전달한다. 테우스는 각 지점에서 플레이어가 가까이 올 때 다음 지점으로 이동하고, HQ 출구에서는 안내 모드를 종료한 뒤 훈련장 시작점으로 함께 이동한다.

`TutorialZoneTransitionHost`는 다음 순서로 단일 씬 구역 전환을 수행한다.

1. 입력 비활성화 및 플레이어 속도 초기화
2. `0.35초` 페이드 아웃
3. 훈련장 Root 활성화
4. 플레이어와 테우스를 훈련장 상단 EntrySpawn으로 이동
5. 카메라 범위 갱신 및 HQ 출구 퀘스트 신호 발행
6. `0.15초` 암전 유지 후 `0.45초` 페이드 인
7. 입력 복구 및 HQ Root 비활성화

수직 구조 검증 결과:

- HQ 브리핑 단상 상단 `+1m` → 메인 홀 `0m` → 출구 전실 `+1m`
- 각 높이 차는 `0.5m` 계단으로 연결돼 점프 없이 이동 가능
- 플레이어 높이 약 `1m`, 단일 점프 상승량 약 `2.45m`
- HQ 바닥부터 프레임 상단까지 `6~7m` 여유가 있어 점프 시 천장 충돌 없음
- 훈련장 입구 상단 약 `+2m` → 훈련장 경기장 바닥 `-4m`
- 훈련장 카메라는 Y `-1.0~0.5` 범위에서 수직 추적해 상단 입구와 하단 경기장 모두 화면 가장자리 쏠림을 방지

플레이 모드에서 HQ 비활성화, 훈련장 활성화, 플레이어 위치 이동, 카메라 범위 전환, 퀘스트 `QST-TUTO-001 → QST-TUTO-004`, 페이드 복구, 입력 복구를 확인했다. 확장된 Tutorial Scene Validator와 EditMode 테스트 21개도 모두 통과했다.

## 13. 훈련장 4구간 재구성

`Z02_TrainingRoom`을 HTML의 훈련 흐름에 맞춰 상단 진입부와 하단의 네 훈련 구간으로 다시 구성했다.

1. **대시 구간** — 바닥 위험 표식 3개를 통과하며 대시 3회 수행
2. **점프 구간** — 높이가 다른 장애물 3개를 넘으며 점프 3회 수행
3. **기본 공격 구간** — `TutorialEnemy`를 배치한 전투 구역에서 공격 3회 수행
4. **펄스 구간** — 교체 가능한 `ART_SLOT` 표적 3개와 모듈 사용 목표 배치

퀘스트 순서는 `QST-TUTO-001 → 004 → 002 → 003 → 005 → 006`으로 연결했다. 각 훈련 경계에는 `TutorialQuestGateHost`를 배치했으며 현재 단계의 조건을 완료해야 다음 구간의 콜라이더와 시각 표시가 해제된다. 모든 배경과 표적은 기능과 분리된 도형 또는 `ART_SLOT`이므로 이후 실제 아트 교체 시 퀘스트 연결을 유지할 수 있다.

훈련장 시작점은 상단 Y `+2.6m`, 주 훈련 바닥은 Y `-4m`로 유지했다. 계단식 하강 후 네 구간은 동일한 하단 기준선에 이어지며, 구간 프레임은 점프 동선을 막지 않는 높이로 배치했다. 훈련장 출구에는 두 번째 `TutorialZoneTransitionHost`를 연결해 완료 후 `Z03_CryonReward`로 페이드 전환한다.

## 14. 훈련장 동적 출현과 Z03 장비 지급

훈련장 대시 퀘스트가 시작되면 `ART_SLOT_FallingCrate_01~03`이 상단 Anchor에서 하단 레인으로 시차를 두고 낙하한다. 기본 공격 퀘스트가 시작되면 바닥 경고 표식이 먼저 켜지고, `TutorialEnemy`가 상단 Spawn Anchor에서 전투 위치로 내려온 뒤 충돌과 공격 기능이 활성화된다. 모든 연출 대상은 미리 배치된 오브젝트이며 실제 아트 교체 시 Host 참조를 유지한다.

`Z03_CryonReward`에는 기존 부츠 픽업과 릴레이 기능을 옮겨 연결했다. 크리온의 장비 패키지는 더블 점프 부츠와 펄스 실전 운용 모듈 시각 슬롯을 함께 가지며, `QST-TUTO-006`은 다음 세 조건을 모두 요구한다.

- 크리온 장비 패키지 수령
- 높은 발판을 향해 실제 더블 점프 1회 사용
- `I` 키로 기본 모듈 트리 열기

장비를 받으면 더블 점프가 영구 저장되고 훈련장에서 임시 사용했던 펄스 모듈이 실전 슬롯에 기록된다. 목표 화살표는 장비 수령 전에는 패키지, 수령 후에는 단일 점프로 닿지 않는 높은 발판을 가리키며, 더블 점프가 확인되면 사라진다. 외부 접근 구간은 별도의 점프맵 없이 평지 직선 통로로 유지하며, 통로 끝의 릴레이를 활성화하면 `QST-TUTO-007`이 완료된다.

## 15. 외곽 전투 스테이지 I

릴레이 활성화 후 `Z03_CryonReward` 출구에서 `Z04_ExteriorCombat_A`로 페이드 전환한다. 퀘스트 순서에는 `QST-TUTO-007-A`와 후속 `QST-TUTO-007-B`를 추가해 두 일반 전투 구역을 보스 퀘스트와 분리했다.

`Z04`는 `TutorialSequentialEncounterHost`가 다음 순서로 진행한다.

1. 스폰 위치에 경고 표식 표시
2. `ExteriorA_Enemy_01_ART_SLOT` 활성화
3. 현재 적 처치 후 다음 위치에 경고 표시
4. 적 2, 적 3을 같은 방식으로 순차 활성화
5. 세 번째 적 처치 시 `ENCOUNTER-A-CLEAR` 신호 발행
6. `CombatA_ExitGate` 콜라이더와 렌더러 비활성화
7. Z04 출구 진입 시 `Z05_ExteriorCombat_B`로 페이드 전환

세 적은 기존 전투 기능을 복제하되 `ENEMY-EXT-A-01~03`의 독립 Stable ID를 사용한다. 동시에 여러 적을 활성화하지 않아 첫 외곽 전투의 난이도를 제어하며, 모든 시각 Root는 후속 아트 교체가 가능하다.

## 16. 외곽 전투 스테이지 II와 보스 구역 진입

`Z05_ExteriorCombat_B`는 `TutorialWaveEncounterHost`를 사용해 Z04보다 높은 전투 밀도를 제공한다.

- **웨이브 1:** 낮은 좌측 발판과 우측 발판에서 적 2기 동시 출현
- **웨이브 2:** 중앙 기둥과 높은 발판에서 강화된 적 2기 동시 출현

각 적은 `ENEMY-EXT-B-01~04`의 독립 ID를 사용한다. 웨이브의 모든 적을 처치해야 다음 웨이브 경고가 시작되며, 두 번째 웨이브 전멸 시 `ENCOUNTER-B-CLEAR` 신호를 발행하고 `CombatB_ExitGate`를 해제한다.

Z05 출구는 `Z06_OreStorage_Boss`의 입구로 페이드 전환한다. 기존 `TutorialHelte`는 Z06의 `GameplayRoot`로 이동해 참조를 유지했으며, 보스 활성 조건은 `QST-TUTO-007-B` 완료로 변경했다. 따라서 두 외곽 전투를 모두 통과하기 전에는 헬테가 활성화되지 않는다.

## 17. Z06 보스전 진입과 패턴 공간

Z06 입구와 실제 전투 공간 사이에 `BossArena_StartTrigger`를 배치했다. 플레이어가 이 지점을 통과하기 전에는 헬테의 본체 충돌, 일반 공격, 보스 패턴이 비활성화되어 입장 페이드 직후 공격받지 않는다.

전투 진입 흐름은 다음과 같다.

1. 플레이어가 저장고 입구에서 보스 경기장 쪽으로 이동
2. 시작 Trigger 통과 시 `BossArena_EntryGate_ART_SLOT` 활성화
3. 보스 경고 슬롯과 좌·중·우 3개 패턴 구역 슬롯을 `0.8초` 동안 표시
4. 경고가 끝나면 헬테의 충돌, 공격, 패턴 활성화
5. 헬테 처치 시 입구 게이트 해제 및 기존 튜토리얼 결과 UI 표시

게이트, 경고, 패턴 구역은 모두 단순 도형 기반 `ART_SLOT`이다. 이후 아트 작업에서는 `TutorialBossArenaHost`의 참조를 유지한 채 렌더링 오브젝트만 교체할 수 있다. 보스 경기장은 세 구역을 넓게 나눠 순간 이동 및 낙하 검 패턴의 회피 방향을 읽을 수 있도록 구성했다.

## 18. 통합 진행 및 HUD 폴리싱

5개 구역 전환은 현재 퀘스트를 추가로 확인하도록 보강했다. 충돌 지형을 우회해 출구 Trigger에 도달하더라도 아래 단계가 아니면 전환되지 않는다.

- Z01 → Z02: `QST-TUTO-001`
- Z02 → Z03: `QST-TUTO-006`
- Z03 → Z04: `QST-TUTO-007-A`
- Z04 → Z05: `QST-TUTO-007-B`
- Z05 → Z06: `QST-TUTO-008`

우측 전투 대상 체력 UI는 현재 활성화된 Actor를 자동 선택한다. HQ와 비전투 구간에서는 숨겨지고, 훈련 적·외곽 전투 적·헬테가 활성화되면 각각 `적` 또는 `헬테` 레이블과 현재 체력을 표시한다. 모델이나 시각 Root를 교체해도 `CombatActorHost` 참조를 유지하면 동작한다.

튜토리얼 완료 결과 화면에서는 목표, 키 안내, 구역 캡션, 상호작용 안내, 플레이어/적 체력, 인벤토리 버튼, 대화 패널과 소개 카드를 숨긴다. 결과 UI와 다음 챕터 이동 안내만 남겨 중복 정보와 화면 겹침을 제거했으며, 강제 완료나 완료 저장 복원 상황에서도 결과 화면만 보이도록 했다.

## 19. 점프 판정 수정과 대화 복원

기존 GroundProbe는 모든 Layer와 Trigger를 검사해 `CameraBounds` 같은 구역 Trigger를 지면으로 오인했다. 이 때문에 공중에서도 착지 상태가 계속 갱신되어 점프가 무제한으로 가능했다. Ground 판정에서 Trigger와 플레이어 자신의 Collider를 제외해 다음 규칙으로 수정했다.

- 부츠 획득 전: 지상 점프 1회
- 부츠 획득 후: 지상 점프 1회 + 공중 추가 점프 1회
- 활공과 구역 Trigger는 점프 횟수를 충전하지 않음

HTML 프로토타입의 대화 흐름을 기준으로 10개 Narrative Beat를 확장했다. HQ에서는 시스템 독백 뒤 에온, 프로메, 아르온, 테우스가 이름이 표시된 대화를 주고받는다. 훈련 안내도 대시·점프·공격·펄스별 3~4줄로 확장했고 현재 Unity 조작키에 맞춰 `LEFT SHIFT`, `SPACE`, `ENTER`, `2`로 수정했다.

다음 구역용 대화가 이전 구역에서 먼저 재생되지 않도록 일부 Beat는 페이드 전환 완료 신호를 기다린다.

- 크리온 장비 대화: `TUTORIAL-TRAINING-EXIT` 이후 Z03에서 재생
- 외곽 전투 I 대화: `TUTORIAL-Z03-EXIT` 이후 Z04에서 재생
- 외곽 전투 II 대화: `TUTORIAL-ENCOUNTER-A-EXIT` 이후 Z05에서 재생
- 헬테 대화: `TUTORIAL-ENCOUNTER-B-EXIT` 이후 Z06에서 재생

대사 문자열의 `화자: 내용` 형식을 Presenter가 분리해 대화창의 이름 영역에는 화자, 본문 영역에는 대사만 표시한다. 모든 등장인물 도형은 기존 `ART_SLOT`을 유지하므로 모델 교체와 대화 시스템은 분리되어 있다.

## 20. 대화 화자 연출과 초상화 슬롯

HQ 대화에 엘륨의 장비 점검 대사를 추가해 에온, 프로메, 엘륨, 아르온, 테우스가 모두 실제 대화 흐름에 참여한다.

`TutorialDialoguePanel`에는 좌우 화자 표시를 연결했다.

- 프로메 발화: 왼쪽 이름 띠와 `Portrait_ART_SLOT` 활성화
- 동료·테우스·크리온·헬테 발화: 오른쪽 이름 띠와 `Portrait_ART_SLOT` 활성화
- 시스템 독백: 양쪽 화자 슬롯 비활성화
- 구역명: 좌우 이름 띠와 겹치지 않도록 상단 중앙에 유지

현재 초상화는 HTML 프로토타입의 인물 색상을 사용한 단순 UI 도형이다. 이후 실제 초상화를 각 `Portrait_ART_SLOT`의 `Image.sprite`에 연결하면 코드가 색상을 덮어쓰지 않으므로 대화 로직을 수정하지 않고 교체할 수 있다.

## 21. 외부 직선 동선과 적 스테이지 연결

외부 접근 구간은 별도의 점프맵으로 구성하지 않는다. `Z03_CryonReward`에서 크리온에게 더블점프 부츠와 모듈을 받은 뒤 짧은 직선 통로의 릴레이를 활성화하고, 즉시 두 개의 외부 적 스테이지로 이어진다.

현재 확정된 외부 진행 순서는 다음과 같다.

1. 크리온 장비 패키지 수령
2. 외부 직선 통로와 릴레이 활성화
3. `Z04_ExteriorCombat_A` 적 3기 순차 전투
4. `Z05_ExteriorCombat_B` 적 4기, 2웨이브 전투
5. 광물 보관소에서 도형 상태의 헬테 조우

외부 대화는 비행선, 헬테의 보안키, 접근하는 외곽 경비를 짧게 설명한다. 이동과 전투를 막지 않는 테우스 자막으로 데미우르고스, 도면, 제니스, 아에테르의 역사, 송신탑을 순서대로 설명한다. 마지막 송신탑 설명은 전투 중이 아니라 Z06의 헬테 접근 복도에서 재생한다. 별도의 외부 점프맵은 두지 않으며, 더블 점프 확인은 Z03의 짧은 장비 시험 발판에서 끝낸다.

헬테는 현재 교체 가능한 도형과 기존 전투 기능을 유지한다. 완성형 보스 조우 연출과 전투 전후 컷신은 이후 별도 단계에서 작업한다.

## 22. 헬테 보스 스테이지 재정리

헬테의 헤르메스 레퍼런스와 고속 이도류·블링크 전투를 읽기 쉽게 만들기 위해 Z06을 안전한 접근로와 평지형 경기장으로 분리했다. 상세 공격 FSM은 후속 명세 단계로 남기고, 이번 단계에서는 지형과 기능 Anchor만 확정했다.

```text
EntrySpawn 948
→ 송신탑 세계관 자막 957
→ 조우/대화 구간 978~986
→ 전투 입구 게이트 987
→ BossArena_StartTrigger 990
→ 평지형 보스 경기장 987~1009
```

- 접근로 바닥: 폭 `47m`, 월드 X `940~987`
- 보스 경기장: 폭 `22m`, 월드 X `987~1009`
- 경기장에는 발판·낙사·환경 피해를 두지 않는다.
- 기존 보스 중앙의 충돌 단상을 제거하고 시각 전용 중앙 표식으로 변경했다.
- 헬테 시작 위치는 `998`로 이동해 전투 Trigger 진입 순간 한 화면 안에 들어오게 했다.
- 좌·중·우 패턴 표식은 `992 / 998 / 1004`로 압축했다.
- Z06 카메라 범위는 X `950~1000`, Fixed Y `1.5`로 조정했다.

`Anchors/HelteStageAnchors`에는 이후 확정 FSM과 연출이 바로 사용할 수 있도록 다음 위치를 사전 배치했다.

- `ApproachCheckpointAnchor`
- `ArenaEntryAnchor`
- `BossDialogueAnchor`
- `BossCenterAnchor`
- `BossCameraFocusAnchor`
- `BossDefeatAnchor`
- `BossBlinkLeftAnchor`, `BossBlinkRightAnchor`
- `SwordSpawn_Left`, `SwordSpawn_Right`, `SwordSpawn_Center`

블링크 FSM은 헬테 자식의 로컬 Anchor가 아니라 경기장에 고정된 좌우 Anchor를 한계값으로 사용한다. 플레이어 좌우 위치를 계산한 뒤 이 범위 안으로 Clamp하므로 경기장 밖으로 순간이동하지 않는다.

## 23. 헬테 전투 FSM 1차 구현

전달된 헬테 명세를 기준으로 임시 타이머 패턴을 상태 기반 전투로 교체했다. 별도의 `EnemyAttackHost` 자동 공격은 보스전 중 비활성화하며, 모든 피해는 헬테 FSM의 명시된 공격 판정에서만 발생한다.

- 1페이즈: 일반 연속베기 1~2회(랜덤) → 블링크 돌진 X베기
- 2페이즈 진입: 체력 50%에서 1회 전환 표식
- 2페이즈: 일반 연속베기 1~2회(랜덤) → 칼 3개 소환 → 블링크 돌진 X베기
- 일반 공격: 첫 베기 → 1m 전진 → 두 번째 베기 → 2초 쿨다운
- 블링크: 기존 위치 잔상 → 플레이어 좌우 3m 중 랜덤 재배치 → 4m 무피해 돌진 → 경고 후 X베기
- 칼 소환: 좌 → 우 → 중앙 순서로 발사하며, 각 칼은 발사 순간의 플레이어 위치를 조준한다.
- 칼 발사가 끝난 뒤 1초 회복 시간을 가진다.

`GameplayRoot/HelteCombatPresentation_ART_SLOTS` 아래에는 잔상, 돌진 경로, X베기 경고, 페이즈 전환, 소환 칼 3개의 교체용 도형을 미리 배치했다. 런타임에는 오브젝트를 새로 생성하지 않고 이 슬롯을 이동·활성화하므로, 이후 모델·VFX를 연결할 때 동일 이름과 직렬화 참조를 유지하면 전투 코드를 수정할 필요가 없다.

## 24. 프로메 3단 근접 콤보와 원거리 공격 기반

프로메의 기본 클릭 공격에 `0.5초` 콤보 입력 창을 적용했다. 실제 공격 쿨다운을 통과한 클릭만 콤보로 인정한다.

- 1단: 기존 하향 베기 형태
- 2단: 반대 방향으로 올려 베는 형태
- 3단: 전방 이동감이 큰 마무리 찌르기 형태
- 각 공격 후 0.5초 안에 다음 공격이 수락되면 다음 단계로 진행한다.
- 3단 이후의 다음 공격 또는 0.5초를 넘긴 입력은 다시 1단으로 시작한다.
- 현재 세 단계의 피해량은 동일하며 모션과 공격 ID만 구분한다. 실제 애니메이션 클립 연결 시 `MeleeAttackHost.ComboStageChanged`의 단계값을 Animator 파라미터로 사용하면 된다.

원거리 공격은 발동 방식이 확정되지 않았으므로 입력에 연결하지 않았다. `PlayerRoot/RangedAttackRoot`에 `RangedProjectileVisual_ART_SLOT`과 충돌 판정, 이동·피해 기능을 사전 배치했으며 `PlayerRangedAttackHost.TryFire()` 또는 `TryFire(Vector2)`를 호출하면 발사된다. 이후 키 입력, 무기 전환, 차지 공격, 모듈 조합 중 하나가 확정되면 호출부만 연결하면 된다.

## 25. 헬테 보스 체력바와 화면 비율 점검

헬테의 실제 전투 판정이 활성화된 시점부터 화면 하단 중앙에 전용 보스 체력바를 표시한다. 전투 전 경고와 대화 중에는 숨기며, 처치 또는 전투 종료 시 다시 숨긴다.

- `BossHealthBarPanel`: 1920×1080 기준 폭 980, 높이 82, 하단 중앙 배치
- `BossHealthBarFill_ART_SLOT`: 이후 실제 보스 UI 아트로 교체할 수 있는 Fill 이미지
- 체력 50% 위치에 2페이즈 구분선 표시
- 색상 외에도 `헬테 현재 체력 / 최대 체력` 숫자를 표시
- 전용 보스 체력바가 표시되는 동안 기존 우측 상단의 간이 적 체력 텍스트는 헬테를 표시하지 않음
- 튜토리얼 결과 화면에서는 다른 HUD와 함께 숨김

현재 카메라 Orthographic Size는 `5`이며 16:9 화면에서 월드 영역 `17.78 × 10`을 표시한다. 프로메 도형 높이는 `1.5`로 화면 높이의 `15%`, 헬테 도형 높이는 `2.5`로 `25%`다. 현재 구성은 환경을 넓게 보여주는 오리 계열보다는 캐릭터와 전투 판독을 우선하는 스컬 계열에 더 가깝다.

중간 지점을 목표로 할 경우 실제 캐릭터 아트 연결 후 Orthographic Size `5.5~5.75`를 1차 후보로 사용한다. 이 범위에서 프로메는 화면 높이의 약 `13~13.6%`가 되고 좌우 시야도 약 `19.6~20.4m`로 넓어진다. 다만 실제 스프라이트의 투명 여백과 공격 이펙트 크기가 확정되기 전에는 카메라 값을 변경하지 않는다.

## 26. UI 상태 우선순위와 겹침 방지

각 UI Presenter가 독립적으로 활성 상태를 변경하더라도 화면 모드에 맞는 HUD만 남도록 `TutorialHudStateCoordinator`를 추가했다. 우선순위는 `Result > Dialogue/Introduction > BossCombat > Normal`이다.

- 대화·인물 소개: 목표, 방향 화살표, 키 안내, 상호작용 안내, 양쪽 체력, 인벤토리 버튼, 구역 캡션을 임시 숨김
- 보스 전투: 상단 목표 패널·방향 화살표·키 안내·상호작용 안내·간이 적 체력·인벤토리 버튼을 숨기고 플레이어 체력과 하단 보스 체력바만 유지
- 결과 화면: 대화, 자막, 목표, 방향 화살표, 체력, 인벤토리, 모듈 패널, 보스 체력바를 모두 숨기고 결과 Overlay만 유지
- 상태 진입 전 각 오브젝트의 활성 상태를 저장하고 상태 종료 시 복원하므로, 조건부 UI를 무조건 켜는 부작용을 피한다.
- 결과·대화·보스 상태가 같은 프레임에 겹칠 경우에도 높은 우선순위 상태가 화면을 소유한다.

## 27. 카메라 가독성 정책

- 이동 속도 방향으로 최대 `2m` Look Ahead를 적용하고 정지 시 중앙으로 복귀한다.
- 수직 추적은 `0.65m` 데드존과 별도 추적 속도를 사용해 점프 시 잔떨림을 줄인다.
- 헬테 전투 중에는 플레이어와 헬테 위치를 함께 계산하며 Orthographic Size를 `5 → 6.25`로 전환한다.
- 피격 흔들림은 최대 `0.2m` 안에서 접근성 Motion Intensity를 곱해 적용한다.

## 28. 전투 가독성 정책

- 일반 적 공격을 `예고 0.28초 → 판정 0.1초 → 후딜`로 분리하고 교체 가능한 `AttackWarning_ART_SLOT`을 사용한다.
- 플레이어 피격 후 `CombatActorHost.hitRecoverySeconds` 동안 실제 중복 피해 방지 무적을 적용한다.
- 헬테 FSM 상태는 보스 체력바 위에 `예고 / 위험 / 기회 / 2 PHASE` 텍스트로 표시해 색상에만 의존하지 않는다.
- 프로메 근접 공격은 활성 Hitbox 시간 동안 공격 방향을 잠그고 판정 종료 후 최신 조준 방향을 반영한다.

## 29. 입력 장치와 조준 정책

- 마우스 공격은 클릭 위치, 게임패드/조이스틱은 오른쪽 스틱과 마지막 이동 방향을 사용한다.
- 게임패드 `A/Cross`는 점프와 홀드 활공, D-pad Up은 모듈 트리, Select은 인벤토리에 연결한다.
- HUD와 대화 진행 안내는 현재 Control Scheme의 실제 Binding Display String을 사용한다.
- 원거리 공격은 기능 슬롯만 유지하며 발동 방식 확정 전까지 입력 Action에 연결하지 않는다.

## 30. 실패와 재시작 정책

- 사망 시 씬을 다시 불러오거나 진행 데이터를 초기화하지 않는다.
- 현재 퀘스트에 대응하는 10개 구역 Entry 체크포인트로 페이드 재배치한다.
- 페이드 동안 입력·이동을 차단하고 모든 전투 RuntimeState를 초기화한다.
- 헬테 재도전은 체력, FSM, 게이트, 경고 슬롯, 보스 UI 상태를 초기화한 뒤 입장 Trigger부터 다시 시작한다.

## 31. 아트 교체 계약

모든 `CombatActorHost`에는 `ArtReplacementContractHost`를 배치한다. 계약은 다음 참조를 기능 코드와 분리해 보존한다.

- 시각 Root와 `FootAnchor_ART_BIND`
- 본체 Collider와 Trigger Attack Hitbox 목록
- Renderer 목록과 예상 Sorting Layer
- Validator의 발 위치 오차 허용값 `0.25m`

실제 모델·스프라이트 교체 후 Validator를 실행하면 발 위치, 누락 Hitbox, Sorting 불일치를 즉시 확인할 수 있다.

## 32. 접근성 기본 프로필

- 카메라 Motion Intensity `0.65`, 강한 점멸 표현용 Intensity `0.45`를 상한으로 사용한다.
- 대화·소개·세계관 자막은 최소 Font Size `20`과 배경 Alpha `0.88`을 보장한다.
- 체력·페이즈·공격 상태는 색상과 텍스트를 함께 사용한다.
- 키 안내는 하드코딩된 키 이름 대신 현재 입력 장치 Binding을 표시한다.
