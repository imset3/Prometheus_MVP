# 09. UI / 진행도 / 결과 구조

## 1. UI 화면 목록

| UI ID | 화면명 | 열기 조건 | 표시 정보 | 관련 시스템 |
| --- | --- | --- | --- | --- |
| UI-001 | HUD | 기본 표시 | HP, 레벨, 경험치, 모듈 슬롯, 쿨타임, 현재 퀘스트 | 전투 / 레벨 / 퀘스트 |
| UI-002 | 모듈 트리 메뉴 | I 입력 또는 메뉴 선택 | 보유 모듈 트리, 해금 가능 모듈, 모듈 포인트 | 모듈 트리 / 모듈 |
| UI-003 | 지도 | Tab 입력 | 현재 스테이지, 목표 위치, 송신탑, 포탈 | 지도 / 퀘스트 |
| UI-004 | 일시정지 메뉴 | Esc 입력 | 재개, 설정, 종료 | 입력 / 설정 |
| UI-005 | 보스 보상 화면 | 보스 처치 후 | 해금된 보스 모듈 트리, 회차 보상 | 전투 / 모듈 트리 |
| UI-006 | 결과 화면 | CHAPTER_01 클리어 또는 회차 종료 | 클리어 시간, 사망 횟수, 피격 횟수, 처치 수 | 저장 / 회차 |

## 2. UI 이벤트 흐름

```txt
게임 이벤트 발생
→ 시스템 이벤트 발행
→ UI Controller가 이벤트 수신
→ HUD / 지도 / 메뉴 표시 갱신
```

## 4. UI 요구사항

| ID | 요구사항 | 설명 | 우선순위 |
| --- | --- | --- | --- |
| UI-REQ-01 | UI는 게임 데이터를 직접 변경하지 않는다. | 시스템 이벤트를 받아 표시만 갱신한다. | 상 |
| UI-REQ-02 | 모듈 트리 UI는 회차/영구 데이터를 구분해 표시한다. | 보유 트리는 영구, 해금/강화/장착은 회차 상태로 표시한다. | 상 |
| UI-REQ-03 | 보스 보상은 영구 해금임을 표시한다. | 보스 모듈 트리 해금 결과를 명확히 보여준다. | 상 |
| UI-REQ-04 | 회차 재시작 시 UI 상태를 초기화한다. | 열린 메뉴, 상태이상, 보스 UI, 퀘스트 표시를 기본 상태로 되돌린다. | 상 |
