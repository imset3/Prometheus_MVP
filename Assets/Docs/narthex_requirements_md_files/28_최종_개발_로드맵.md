# 28. 최종 개발 로드맵

## 1. 최종 기준

이 문서는 실제 Unity 6.3 URP 개발 착수를 위한 최종 로드맵이다.

기존 [[25_개발_로드맵.md]]는 큰 흐름이고, 이 문서는 가상 구현 시뮬레이션에서 발견한 Unity 문제까지 반영한 최종 실행 순서이다.

## 2. 절대 원칙

| 원칙 | 내용 |
| --- | --- |
| 콘텐츠보다 기반 먼저 | Boot, Registry, Save, SceneFlow, Validator, 테스트 전에는 콘텐츠 제작을 시작하지 않는다. |
| Definition은 읽기 전용 | 런타임 변경은 RuntimeState에만 기록한다. |
| 저장은 단순 DTO | Dictionary, UnityEngine.Object, SceneAsset, 인터페이스 저장 금지 |
| 무거운 리소스는 Addressables | Prefab, Scene, VFX, Audio는 LoadScope로 로드/해제한다. |
| Gameplay는 Physics2D | 시각 표현이 3D여도 전투/이동 판정은 Physics2D로 고정한다. |
| 기본 Renderer는 URP 2D Renderer | Universal Renderer는 별도 RendererProfile로 승인된 3D 연출 씬에서만 사용한다. |
| 스테이지는 Stage/Room 데이터로 제작 | StageRoot, StageDefinition, RoomDefinition, CameraBounds, PortalRoot를 같은 흐름으로 연결한다. |
| 전투 좌표계는 X/Y 고정 | Z축은 렌더링 정렬 전용이며 Hitbox, Hurtbox, Projectile 판정에 사용하지 않는다. |
| UI는 uGUI + TextMeshPro | 런타임 UI Toolkit 혼용 금지 |
| 검증 실패 시 진행 금지 | Validator 실패 상태에서 콘텐츠 제작 또는 빌드 금지 |

## 3. Milestone 0 - 프로젝트 고정

| 작업 | 산출물 | 통과 기준 |
| --- | --- | --- |
| Unity 6.3 URP 프로젝트 생성 | 빈 URP 프로젝트 | URP Asset이 Graphics/Quality에 연결 |
| 필수 패키지 고정 | Input System, Addressables, Cinemachine, TMP, Test Framework | Package Manager에서 설치 확인 |
| RendererProfile 고정 | RENDERPROFILE-DEFAULT-2D | 기본 씬이 URP 2D Renderer 사용 |
| 폴더 구조 생성 | `Assets/_Project`, `ThirdParty`, `AddressableAssetsData` | 26번 문서 4장과 일치 |
| asmdef 생성 | Game.Core 등 | 순환 참조 없음 |
| 씬 생성 | Boot, Loading, Title | Boot/Loading은 Build Settings 포함 |

## 18. 최종 결론

이 로드맵 기준이면 지금 바로 개발을 시작해도 된다.

실제 Unity에서 문제가 될 수 있는 지점은 [[27_가상_구현_시뮬레이션_검토.md]]에 반영했고, 그 방지책을 이 로드맵의 Milestone 0~7에 배치했다.

2D 액션 플랫포머 제작 중 흔들릴 수 있는 렌더러, 스테이지, 카메라, 이동 감각, 히트박스, 보스 아레나 기준은 [[29_2D_액션_최적화_구조_검증.md]]에 반영했고, 그 검증 항목을 이 로드맵의 Milestone 0, 2, 5, 6, 8~10과 최종 Gate에 배치했다.

최종 보스는 후반 콘텐츠 입력으로 남겨도 구조적으로 안전하다.
