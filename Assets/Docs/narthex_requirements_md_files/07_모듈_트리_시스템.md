# 07. 모듈 트리 시스템

## 1. 모듈 트리 시스템 정의

모듈 트리는 플레이어가 회차마다 사용할 모듈을 해금, 강화, 장착하는 성장 구조이다.

이전 성장 시스템은 사용하지 않는다. 해당 역할은 기본 모듈 트리와 보스 모듈 트리로 대체한다.

플레이어는 게임 시작 시 기본 모듈 트리에 접근할 수 있다.

보스 처치 시 해당 보스의 보스 모듈 트리가 영구 해금된다.

모듈 트리 접근 권한은 영구 데이터로 유지되지만, 트리 내부의 모듈 해금/강화/장착 상태와 모듈 포인트는 회차 데이터로 관리한다.

## 2. 기본 구조

```txt
보유 모듈 트리 목록 (영구 데이터)
├─ 기본 모듈 트리
└─ 보스 모듈 트리
   ├─ 튜토리얼 보스 트리
   ├─ CHAPTER_01 중간 보스 트리
   └─ CHAPTER_01 최종 보스 트리

회차 모듈 빌드 (회차 데이터)
├─ 모듈 포인트
├─ 해금 모듈
├─ 강화 단계
└─ 장착 슬롯
```

## 3. 모듈 트리 종류

| 트리 종류 | 획득 방식 | 유지 여부 | 설명 | 우선순위 |
| --- | --- | --- | --- | --- |
| 기본 모듈 트리 | 신규 게임 시작 시 기본 제공 | 영구 유지 | 플레이어가 처음부터 접근 가능한 기본 성장 트리 | 상 |
| 튜토리얼 보스 트리 | BOSS-TUTO-HELTE 처치 | 영구 유지 | 헬테의 블링크/고속 이동 테마 보스 모듈 트리 | 중 |
| CHAPTER_01 중간 보스 트리 | BOSS-CH01-MID-MINOTAUR 처치 | 영구 유지 | 미노타우르스의 돌진/충격 테마 보스 모듈 트리 | 상 |
| CHAPTER_01 최종 보스 트리 | BOSS-CH01-FINAL 처치 | 영구 유지 | 최종 보스 입력 후 테마 확정 | 중 |

## 4. 모듈 트리 규칙

| ID | 규칙 | 설명 | 우선순위 |
| --- | --- | --- | --- |
| TREE-RULE-01 | 기본 모듈 트리는 처음부터 접근 가능하다. | 신규 게임 시작 시 기본 모듈 트리 접근 권한을 가진다. | 상 |
| TREE-RULE-02 | 보스 모듈 트리는 보스 처치로 해금한다. | 보스 처치 결과가 저장되면 해당 보스 트리가 영구 해금된다. | 상 |
| TREE-RULE-03 | 트리 접근 권한은 영구 유지된다. | 사망 또는 회차 재시작 후에도 보유 모듈 트리 목록은 유지된다. | 상 |
| TREE-RULE-04 | 모듈 포인트는 회차 자원이다. | 회차 중 획득하고 사망 시 초기화한다. | 상 |
| TREE-RULE-05 | 모듈 해금 상태는 회차 데이터이다. | 각 회차에서 다시 해금해야 하며 사망 시 초기화한다. | 상 |
| TREE-RULE-06 | 모듈 강화 상태는 회차 데이터이다. | 강화 단계는 해당 회차 안에서만 유지한다. | 상 |
| TREE-RULE-07 | 모듈 장착 상태는 회차 데이터이다. | 장착 슬롯 구성은 회차 재시작 시 초기화한다. | 상 |
| TREE-RULE-08 | 해금되지 않은 모듈은 장착할 수 없다. | 모듈 포인트로 해금한 모듈만 슬롯에 장착할 수 있다. | 상 |
| TREE-RULE-09 | 강화 비용은 기본 1이다. | 1강, 2강, 3강에 필요한 모듈 포인트는 각각 1로 시작한다. | 중 |
| TREE-RULE-10 | 최대 강화 단계는 3강이다. | 모듈은 3강을 초과하여 강화할 수 없다. | 상 |

## 5. 착용 슬롯 구조

| 슬롯 종류 | 슬롯 수 | 장착 가능 모듈 | 설명 |
| --- | --- | --- | --- |
| 일반 스킬 슬롯 | 2개 | 근거리, 원거리, 이동 모듈 | 플레이어 입력으로 직접 사용하는 모듈 |
| 확장 스킬 슬롯 | 4개 | 패시브, 강화 모듈 | 착용 중 자동 적용되거나 일반 스킬을 강화하는 모듈 |

## 6. 모듈 상태 데이터

모듈 상태는 하나의 상태 문자열로 합치지 않고 세 축으로 분리한다.

| 축 | 값 | 저장 구분 | 설명 |
| --- | --- | --- | --- |
| 해금 여부 | Locked / Unlocked | 회차 데이터 | 해당 회차에서 모듈을 사용할 수 있는지 |
| 장착 여부 | Unequipped / Equipped | 회차 데이터 | 슬롯에 장착되어 효과가 적용되는지 |
| 강화 단계 | 0 / 1 / 2 / 3 | 회차 데이터 | 해당 회차에서 강화된 단계 |

## 7. 모듈 트리 데이터 구조

| 데이터 항목 | 설명 | 예시 |
| --- | --- | --- |
| 트리 ID | 모듈 트리를 구분하는 ID | TREE-BASIC-001 |
| 트리명 | 표시 이름 | 기본 모듈 트리 |
| 트리 종류 | Basic / Boss | Basic |

| 포함 모듈 ID 목록 | 트리에 속한 모듈 목록 | MOD-SKL-001, MOD-SKL-002 |
| 영구 유지 여부 | 사망 후에도 트리 접근 권한을 유지하는지 | true |

## 7.1 Unity ModuleTreeDefinition 구현 기준

모듈 트리는 `ModuleTreeDefinition`으로 정의하고, 실제 회차 상태는 `RunModuleBuildState`로 분리한다.

| 구현 요소 | 역할 | 저장 구분 |
| --- | --- | --- |
| ModuleTreeDefinition | 트리 ID, 표시명, 트리 종류, 포함 모듈 목록 정의 | 저장하지 않음 |
| ModuleNodeDefinition | 트리 안의 노드 위치, 선행 모듈, 해금 비용 정의 | 저장하지 않음 |
| OwnedModuleTreeState | 접근 가능한 트리 ID 목록 | 영구 데이터 |
| RunModuleBuildState | 모듈 포인트, 해금 모듈, 강화 단계, 장착 슬롯 | 회차 데이터 |

트리 UI는 ModuleTreeDefinition과 RunModuleBuildState를 함께 읽어 표시한다. UI는 모듈 해금을 직접 처리하지 않고 ModuleTreeManager에 해금 요청 이벤트만 전달한다.

## 10. 구현 기준 요약

```txt
모듈 트리 접근 권한 = 영구 데이터
보스 모듈 트리 해금 = 보스 처치 보상
모듈 포인트 = 회차 자원
모듈 해금/강화/장착 = 회차 데이터
회차 재시작 = 모듈 포인트와 회차 빌드 초기화
```
