# 25. 개발 로드맵

## 1. 로드맵 기준

이 로드맵은 [[26_즉시_개발_착수_구조.md]]를 실행 기준으로 삼아, 튜토리얼과 CHAPTER_01을 Unity 6.3 URP에서 확장 가능한 방식으로 구현하기 위한 순서를 정의한다.

실제 구현 순서와 Gate의 최종 기준은 [[28_최종_개발_로드맵.md]]를 따른다.

최종 보스 상세 패턴은 개발 후반까지 입력 대기 상태로 유지한다.

## 2. Phase 0 - 프로젝트 기반 고정

| 목표 | 산출물 | 완료 기준 |
| --- | --- | --- |
| 즉시 착수 기준 적용 | [[26_즉시_개발_착수_구조.md]]의 첫 커밋 구조 | 폴더, asmdef, Boot/Loading/Title 씬 생성 |
| Unity 6.3 URP 기준 고정 | URP Asset, Renderer Asset, Quality 설정 | 모든 Quality Level에서 URP Asset 누락 없음 |
| 폴더 구조 생성 | `Assets/_Project`, `ThirdParty`, `AddressableAssetsData` | 외부 에셋 원본 직접 참조 없음 |
| asmdef 분리 | Game.Core, Content, Save, SceneFlow, Gameplay, Presentation, Editor, Tests | 순환 참조 없음 |
| Addressables 기본 그룹 생성 | Core, Tutorial, Chapter01, Boss, UI, Audio | 중복 address 없음 |
| Validator 골격 생성 | ID 중복, 누락 참조, URP Material 검사 | 빌드 전 수동 실행 가능 |

## 3. Phase 1 - Core / Save / SceneFlow

| 목표 | 산출물 | 완료 기준 |
| --- | --- | --- |
| 앱 부트스트랩 | Boot Scene, GameBootstrap | 앱 시작 후 Registry 로드 성공 |
| 게임 상태 머신 | Boot, Title, Loading, InRun, Paused, Result | 상태 전환 로그 확인 |
| 이벤트 버스 | 타입 기반 GameEventBus | 주요 이벤트 payload 검증 |
| 저장 시스템 | PermanentData, RunData, SettingsData | 신규/로드/저장/초기화 동작 확인 |
| 마이그레이션 | saveVersion, MigrationTable | oldId → newId 변환 테스트 통과 |
| 씬 전환 | SceneFlowManager, Loading 화면 | 포탈 이동과 회차 재시작이 같은 경로 사용 |

## 4. Phase 2 - 데이터 주도 실행 골격

| 목표 | 산출물 | 완료 기준 |
| --- | --- | --- |
| ContentRegistry | Definition 목록과 검색 API | stableId 조회 실패 시 명확한 오류 |
| AbilityExecutor | Wait, Move, SpawnHitbox, SpawnProjectile, PlaySfx Action | 모듈/보스가 같은 Executor 사용 |
| EffectSystem | Damage, Knockback, Stun, StatModifier | 중첩/저항/만료 테스트 통과 |
| CombatSystem | Hitbox, Projectile, DamageCalculator | 대시 무적, 중복 명중 제한 확인 |
| ModuleSystem | 해금, 장착, 강화, 쿨타임 | 회차 재시작 시 초기화 확인 |
| QuestSystem | Condition/Reward 기반 진행 | BossKilled 보상 연결 확인 |

## 5. Phase 3 - Vertical Slice

| 목표 | 산출물 | 완료 기준 |
| --- | --- | --- |
| 플레이어 조작 | 이동, 점프, 더블 점프, 활공, 대시 | 입력 누수 없이 플레이 가능 |
| 기본 전투 | 일반 공격, 피격, 사망 | PlayerDead 이벤트 발생 |

| UI 최소 세트 | HUD, 모듈 트리, 지도, 일시정지 | UI 열림 중 Gameplay 입력 차단 |
| 저장/회차 루프 | 사망 후 STG-001 재시작 | 영구 트리 유지, 회차 데이터 초기화 |
| 검증 테스트 | EditMode + PlayMode 핵심 테스트 | CI 또는 수동 전체 통과 |

## 11. Phase 9 - 폴리싱 / 안정화

| 목표 | 산출물 | 완료 기준 |
| --- | --- | --- |
| 성능 최적화 | Pooling, Addressables Release, GC 확인 | 보스전 프레임 드랍 없음 |

## 12. 구현 우선순위 요약

```txt
0. Unity 6.3 URP / 폴더 / asmdef / Addressables / Validator
1. Boot / GameStateMachine / Save / SceneFlow
2. Definition / Runtime / Executor 실행 골격
3. Player / Combat / Module / Effect / Quest Vertical Slice
4. Tutorial + Helte
5. CHAPTER_01 STG-001~STG-003
6. Minotaur + STG-004
7. STG-005~STG-006 Placeholder + Result
8. Final Boss 후반 통합
9. Polish / Build / Regression
```
