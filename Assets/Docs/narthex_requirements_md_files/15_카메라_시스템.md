# 15. 카메라 시스템

## 1. 카메라 시스템 정의

카메라 시스템은 플레이어의 이동, 전투, 보스전, 지도 확인, 연출 상황에 따라 화면 위치와 범위를 제어한다.

회차 재시작 시 카메라 상태는 STG-001 기본 Follow 상태로 초기화한다.

보스 패턴 상세와 관계없이 보스전 진입/처치/연출 시 카메라 상태 전환은 사용할 수 있다.

## 2. 카메라 상태

| 상태 ID | 상태명 | 설명 | 사용 상황 |
| --- | --- | --- | --- |
| CAM-STATE-01 | Follow | 플레이어 추적 | 일반 탐색 |
| CAM-STATE-02 | CombatFollow | 전투 중 플레이어 추적 | 일반 전투 구역 |
| CAM-STATE-03 | BossCamera | 플레이어와 보스를 함께 보여주는 상태 | 보스전 |
| CAM-STATE-04 | Cinemachine | 연출 카메라 | 프롤로그, 보스 등장, 클리어 연출 |
| CAM-STATE-05 | UIHold | UI 표시 중 카메라 유지 | 지도, 일시정지, 결과 화면 |

## 3. 카메라 기본 규칙

| ID | 규칙 | 설명 | 우선순위 |
| --- | --- | --- | --- |
| CAM-RULE-01 | 기본 카메라는 플레이어 추적 방식이다. | 일반 탐색과 전투 중 플레이어를 중심으로 이동한다. | 상 |
| CAM-RULE-02 | 카메라는 맵 경계를 넘지 않는다. | 현재 스테이지의 제한 영역 밖을 보여주지 않는다. | 상 |
| CAM-RULE-03 | 지도 UI가 열려도 카메라 위치는 변경하지 않는다. | 지도는 UI 기능이다. | 상 |
| CAM-RULE-04 | 회차 재시작 시 초기화한다. | STG-001 기본 Follow 상태를 적용한다. | 상 |
| CAM-RULE-05 | 화면 흔들림은 설정값을 따른다. | 설정에서 OFF이면 흔들림을 출력하지 않는다. | 중 |

## 4. 상태 전환 규칙

| ID | 현재 상태 | 전환 조건 | 다음 상태 | 우선순위 |
| --- | --- | --- | --- | --- |
| CAM-TRANS-01 | Follow | ENEMY 전투 구역 진입 | CombatFollow | 중 |
| CAM-TRANS-02 | CombatFollow | 전투 구역 종료 | Follow | 중 |
| CAM-TRANS-03 | Follow / CombatFollow | 보스전 구역 진입 | BossCamera | 상 |
| CAM-TRANS-04 | BossCamera | 보스 처치 또는 보스전 종료 | Follow 또는 Cinemachine | 상 |
| CAM-TRANS-05 | Follow | 연출 시작 | Cinemachine | 상 |
| CAM-TRANS-06 | Cinemachine | 연출 종료 | 이전 상태 또는 Follow | 상 |
| CAM-TRANS-07 | 모든 플레이 상태 | 지도 또는 일시정지 UI 열기 | UIHold | 중 |
| CAM-TRANS-08 | UIHold | UI 닫기 | 이전 상태 | 중 |
