# 18. 회차 재시작 시스템

## 1. 회차 재시작 시스템 정의

회차 재시작 시스템은 플레이어가 사망했을 때 현재 회차 진행을 종료하고, 영구 데이터만 유지한 채 CHAPTER_01 STG-001 시작 지점에서 새 회차를 시작하는 시스템이다.

송신탑은 저장/진행 보조 장치이며 사망 후 부활 지점으로 사용하지 않는다.

보스 처치로 해금한 보스 모듈 트리 접근 권한은 사망 후에도 유지한다.

레벨, 경험치, 모듈 포인트, 모듈 해금/강화/장착 상태, 퀘스트 진행, 스테이지 진행, 송신탑 상태는 회차 재시작 시 초기화한다.

## 2. 회차 재시작 기본 규칙

| ID | 규칙 | 설명 | 우선순위 |
| --- | --- | --- | --- |
| LOOP-RULE-01 | 사망 시 현재 회차를 종료한다. | HP가 0 이하가 되면 회차 종료 처리를 시작한다. | 상 |
| LOOP-RULE-02 | 송신탑에서 부활하지 않는다. | 송신탑 위치는 사망 후 재시작 위치에 영향을 주지 않는다. | 상 |
| LOOP-RULE-03 | CHAPTER_01 STG-001부터 다시 시작한다. | 튜토리얼 완료 이후 회차 사망은 항상 STG-001로 돌아간다. | 상 |
| LOOP-RULE-04 | 영구 해금 데이터는 유지한다. | 보유 모듈 트리 목록과 보스 모듈 트리 해금 기록은 유지된다. | 상 |
| LOOP-RULE-05 | 회차 성장 데이터는 초기화한다. | 레벨, 경험치, 모듈 포인트, 모듈 해금/강화/장착은 초기화된다. | 상 |
| LOOP-RULE-06 | 진행 상태는 초기화한다. | 스테이지 클리어, 퀘스트, 송신탑, ENEMY, 보스 상태는 초기화된다. | 상 |
| LOOP-RULE-07 | 설정 데이터는 유지한다. | 사운드, 화면, 키 설정은 회차 재시작의 영향을 받지 않는다. | 상 |

## 3. 사망 후 처리 흐름

```txt
플레이어 HP 0 이하
→ 전투 시스템이 플레이어 사망 이벤트 발생
→ 사망 횟수 기록
→ 현재 회차 종료
→ 영구 데이터와 회차 데이터 분리
→ 보유 모듈 트리 목록 유지
→ 보스 모듈 트리 해금 기록 유지
→ 레벨 / 경험치 / 모듈 포인트 초기화
→ 모듈 해금 / 강화 / 장착 상태 초기화
→ 스테이지 / 퀘스트 / 송신탑 / ENEMY / 보스 상태 초기화
→ CHAPTER_01 STG-001 시작 지점 로드
→ 기본 카메라 / 기본 UI / 기본 플레이어 상태 적용
→ 새 회차 시작
```

## 4. 재시작 위치 기준

| 우선순위 | 재시작 위치 | 조건 | 설명 |
| --- | --- | --- | --- |
| 1 | CHAPTER_01 STG-001 시작 지점 | 튜토리얼 완료 후 회차 사망 시 | 기본 재시작 위치 |
| 2 | 튜토리얼 시작 지점 | 튜토리얼 중 전용 재시작이 필요한 경우 | 튜토리얼 예외 |
| - | 송신탑 위치 | 사용하지 않음 | 송신탑은 부활 위치가 아니다. |

## 5. 유지 데이터

| 유지 항목 | 설명 | 우선순위 |
| --- | --- | --- |
| 기본 모듈 트리 접근 권한 | 신규 게임 시작 시 기본 제공되는 트리 | 상 |
| 보스 모듈 트리 해금 기록 | 보스 처치로 얻은 모듈 트리 접근 권한 | 상 |
| 튜토리얼 완료 여부 | 튜토리얼을 다시 진행할지 판단하는 데이터 | 중 |
| 누적 사망 횟수 | 전체 플레이 기록 | 중 |
| 설정 데이터 | 사운드, 화면, 입력 설정 | 상 |

## 7. 모듈 트리와 회차 재구성

| ID | 규칙 | 설명 | 우선순위 |
| --- | --- | --- | --- |
| LOOP-TREE-01 | 보유 모듈 트리 목록은 유지된다. | 사망 후에도 기본 트리와 해금된 보스 트리에 접근할 수 있다. | 상 |
| LOOP-TREE-02 | 모듈 해금은 초기화된다. | 트리 접근 권한은 유지하지만 내부 모듈은 회차마다 다시 해금한다. | 상 |
| LOOP-TREE-03 | 모듈 강화는 초기화된다. | 강화 단계는 회차 빌드 데이터로 취급한다. | 상 |
| LOOP-TREE-04 | 모듈 장착은 초기화된다. | 새 회차에서 다시 빌드를 구성한다. | 상 |
| LOOP-TREE-05 | 모듈 포인트는 초기화된다. | 회차 시작 기본 지급량 또는 0으로 시작한다. | 상 |

| LOOP-CHK-02 | 저장 기능 | 회차 중 진행 보조 저장만 수행한다. | 상 |
| LOOP-CHK-03 | 상태 유지 | 회차 재시작 시 초기화한다. | 상 |
| LOOP-CHK-04 | ENEMY 버프 | 송신탑 상태 초기화와 함께 기본 상태로 돌아간다. | 상 |
| LOOP-CHK-05 | 재시작 위치 영향 | 송신탑은 재시작 위치에 영향을 주지 않는다. | 상 |
