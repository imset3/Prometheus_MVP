# 20. 포탈 시스템

## 1. 포탈 정의

포탈은 현재 스테이지 클리어 조건을 달성한 뒤 다음 스테이지로 이동하기 위해 사용하는 스테이지 이동 장치이다.

플레이어는 포탈 상호작용 범위 안에서 F 키를 눌러 다음 스테이지로 이동할 수 있다.

포탈 상태는 회차 데이터이며 회차 재시작 시 초기화된다.

## 2. 포탈 기본 규칙

| ID | 규칙 | 설명 | 우선순위 |
| --- | --- | --- | --- |
| PORTAL-RULE-01 | 포탈은 다음 스테이지 이동 기능을 가진다. | 현재 스테이지에서 다음 스테이지로만 이동한다. | 상 |
| PORTAL-RULE-02 | 스테이지 클리어 후 활성화된다. | 퀘스트 또는 스테이지 클리어 조건 달성 후 사용할 수 있다. | 상 |
| PORTAL-RULE-03 | 상호작용 입력으로 이동한다. | 접근 또는 충돌만으로는 이동하지 않는다. | 상 |
| PORTAL-RULE-04 | 이동 후 이전 스테이지로 돌아갈 수 없다. | CHAPTER_01은 순차 진행으로 처리한다. | 상 |
| PORTAL-RULE-05 | 이동 전 회차 진행 상태를 정산한다. | 보상, 경험치, 퀘스트, 저장 처리를 연결한다. | 상 |
| PORTAL-RULE-06 | 회차 재시작 시 초기화된다. | 모든 포탈 상태는 초기 상태로 돌아간다. | 상 |

## 3. 포탈 요구사항

| ID | 기능 | 요구사항 | 우선순위 |
| --- | --- | --- | --- |
| PORTAL-REQ-01 | 포탈 배치 | 각 스테이지 완료 지점에 포탈을 배치할 수 있어야 한다. | 상 |
| PORTAL-REQ-02 | 포탈 활성화 | 스테이지 클리어 조건 달성 시 활성화되어야 한다. | 상 |
| PORTAL-REQ-03 | 이동 가능 안내 | 활성 포탈 접근 시 이동 안내 UI를 표시한다. | 중 |
| PORTAL-REQ-04 | 이동 불가 처리 | 비활성 포탈에서 F 입력 시 조건 미달 안내를 표시한다. | 중 |
| PORTAL-REQ-05 | 중복 이동 방지 | 이동 처리 중 동일 포탈 요청을 중복 처리하지 않는다. | 상 |
| PORTAL-REQ-06 | 이동 후 위치 지정 | 다음 스테이지 시작 위치에 플레이어를 배치한다. | 상 |
| PORTAL-REQ-07 | 이동 후 UI 갱신 | HUD, 지도, 퀘스트 목표를 새 스테이지 기준으로 갱신한다. | 상 |

## 4. 데이터 구조

| 데이터 항목 | 설명 | 예시 |
| --- | --- | --- |
| 포탈 ID | 포탈 고유 ID | PORTAL-STG-001 |
| 현재 스테이지 | 포탈이 배치된 스테이지 | STG-001 |
| 이동 대상 스테이지 | 이동할 다음 스테이지 | STG-002 |
| 포탈 상태 | 비활성 / 활성 / 이동 처리 중 / 사용 완료 | 활성 |
| 활성화 조건 | 포탈 활성화 조건 | QST-STG-001 완료 |
| 실행 입력 | 이동 입력 | F |

## 5. 연결 이벤트

| 이벤트 | 발생 시점 | 수신 시스템 |
| --- | --- | --- |
| PortalActivated | 스테이지 클리어 조건 달성 | UI / 지도 / 사운드 |
| PortalUsed | F 입력으로 이동 성공 | 저장 / 퀘스트 / 카메라 / 사운드 |
| PortalReset | 회차 재시작 | 저장 / 지도 |
