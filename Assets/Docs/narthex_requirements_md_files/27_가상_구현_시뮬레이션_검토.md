# 27. 가상 구현 시뮬레이션 검토

## 1. 목적

이 문서는 Codex가 실제로 Unity 6.3 URP 프로젝트에서 코드를 작성한다고 가정하고, 컴파일/직렬화/씬 전환/Addressables/입력/저장/테스트 단계에서 발생할 수 있는 문제를 사전에 검토한다.

## 2. 최종 판정

현재 구조는 개발 착수 가능하다.

다만 Unity 특유의 함정은 존재한다. 아래 항목은 모두 [[26_즉시_개발_착수_구조.md]]와 [[28_최종_개발_로드맵.md]]에 반영해야 하며, 반영하지 않으면 실제 구현 중 문제가 발생한다.

## 3. 가상 구현 시뮬레이션 결과

| 단계 | 내가 작성할 코드 | 발생 가능한 Unity 문제 | 판정 | 조치 |
| --- | --- | --- | --- | --- |
| 프로젝트 생성 | URP 프로젝트, 패키지 설치 | Input System/Addressables/Cinemachine/Test Framework 누락 | 위험 | Phase 0 Package Gate 추가 |
| 폴더/asmdef | Game.Core, Content, Save 등 생성 | asmdef 순환 참조, Editor API Runtime 유입 | 위험 | AsmdefCycleValidator, RuntimeEditorApiValidator 필수 |
| DefinitionBase | ScriptableObject stableId | 런타임에서 Definition 값을 수정할 위험 | 위험 | Definition 읽기 전용, RuntimeState 복사 |
| ActionDefinition | Ability 실행 노드 작성 | Unity가 인터페이스/추상 클래스/다형성 리스트 직렬화에 약함 | 위험 | ActionDefinition을 독립 ScriptableObject Asset으로 시작 |
| SceneDefinition | 씬 참조 작성 | Runtime asmdef에서 SceneAsset 사용 시 UnityEditor 참조 필요 | 위험 | 런타임은 sceneAddress 또는 AssetReference만 사용 |
| SaveData | RunSaveData 작성 | JsonUtility는 Dictionary/다형성 저장에 부적합 | 위험 | Save DTO는 List Pair와 단순 타입만 사용 |
| SaveSystem | 파일 저장 | 저장 중 실패하면 기존 세이브 손상 가능 | 위험 | temp 파일 작성 후 교체, 실패 시 원본 유지 |
| EventBus | 타입 기반 이벤트 | 정적 이벤트 구독 누수, Domain Reload OFF에서 중복 호출 | 위험 | ServiceRoot 수명 이벤트 버스, 정적 이벤트 금지 |
| Input | Gameplay/UI Action Map | UI 중 Gameplay 입력 누수 | 위험 | InputModeController가 State별 Action Map 단독 활성화 |
| Physics | Hitbox/Projectile | 2D/3D Collider 혼용, Trigger 미발생 | 위험 | Physics2D 고정, Rigidbody2D 요구 |
| Animator | 보스 공격 타이밍 | Animator Event 의존 시 데이터 주도 판정과 충돌 | 위험 | 판정은 ActionDefinition 타임라인 소유 |
| Addressables | Stage/Boss 로드 | handle 해제 누락, 씬 언로드 순서 오류 | 위험 | AddressableLoadScope로 소유권 추적 |
| URP | VFX/Material | Built-in/HDRP Shader 유입, Compatibility Mode 의존 | 위험 | URPMaterialValidator 필수 |
| Tests | PlayMode 테스트 | Addressables 비동기 테스트 불안정 | 관리 가능 | Preload 완료 후 테스트 시작 |

## 4. 코드 작성 시 즉시 터질 수 있는 세부 문제

| 문제 | 원인 | 방지 기준 |
| --- | --- | --- |
| `UnityEditor.SceneAsset` 컴파일 오류 | Runtime asmdef에서 Editor 타입 사용 | SceneDefinition 런타임 필드는 string address |
| 저장 파일이 비어 있거나 일부만 저장 | 저장 중 앱 종료 또는 예외 | temp 저장 후 atomic replace |
| 모듈 강화 Dictionary가 저장되지 않음 | Unity JsonUtility Dictionary 미지원 | `List<ModuleUpgradeSaveEntry>` 사용 |
| Ability Action 목록이 Inspector에서 깨짐 | 인터페이스/추상 클래스 리스트 직렬화 | ActionDefinition Asset 참조 리스트 사용 |
| PlayMode 재실행 시 이벤트가 두 번 발생 | 정적 이벤트 구독 잔존 | 정적 이벤트 금지, ServiceRoot Dispose |
| 보스 돌진 Trigger가 안 잡힘 | Collider2D만 있고 Rigidbody2D 없음 | Hitbox/Projectile에 Kinematic Rigidbody2D |
| Pause 중 쿨타임이 계속 돎 | Time.time 직접 사용 | GameClock의 scaled/unscaled 시간 사용 |
| 씬 전환 후 BGM/VFX가 남음 | Addressables handle 미해제 | LoadScope owner별 Release |
| UI 버튼에 한글 문자열이 직접 박힘 | textKey 구조 누락 | TextDefinition 경유 |
| 최종 보스 placeholder가 Validator 실패 | 패턴 미정 슬롯을 오류로 처리 | 명시적 Placeholder AllowList |

## 5. 가상 코드 구조 보정안

```txt
DefinitionBase : ScriptableObject
→ stableId
→ displayNameTextKey
→ editorOnlyMemo

RuntimeState
→ Definition stableId 참조
→ 런타임 변경값만 보유

SaveData DTO
→ string id
→ int/float/bool/string/list only
→ UnityEngine.Object 저장 금지

Executor
→ Definition 읽기
→ RuntimeState 변경
→ EventBus 발행
```

## 6. 구현 전 필수 결정

| 항목 | 결정 |
| --- | --- |
| Gameplay Physics | Physics2D 고정 |
| Runtime UI | uGUI + TextMeshPro |
| Editor UI | Unity Editor IMGUI 또는 UI Toolkit 사용 가능 |
| Definition Action 방식 | ScriptableObject ActionDefinition Asset 방식 |
| Save 직렬화 | 단순 DTO + List Pair 방식 |
| EventBus | ServiceRoot 소유 인스턴스 방식 |
| Addressables | AddressableLoadScope 소유권 방식 |
| Final Boss | Placeholder AllowList로 빌드 허용 |

## 7. 보완 후 판정

위 보정안을 반영하면 Unity에서 바로 코드를 작성해도 구조적 차단 문제는 없다.

남는 위험은 밸런스, 연출 품질, 최종 보스 콘텐츠 품질이며, 이는 구조 문제가 아니라 콘텐츠 제작 단계의 문제다.
