# 16. 전투 시스템

## 1. 전투 시스템 정의

전투 시스템은 플레이어, ENEMY, 보스가 공격을 사용하고, 공격 판정에 명중한 대상에게 피해, 피격 반응, 상태이상, 사망 처리를 적용하는 시스템이다.

전투는 일반 공격, 방향 공격, 모듈(스킬), ENEMY 공격, 보스 공격을 기준으로 발생한다.

보스 패턴 상세는 [[10_ENEMY_BOSS_설계.md]]의 Pattern ID를 원장으로 사용한다.

모듈 사용은 플레이어 입력으로 요청되고 데우스(펫)가 발동 위치와 연출을 담당하며, 전투 시스템은 판정 생성과 명중 처리를 담당한다.

플레이어가 사망하면 회차 재시작 시스템으로 사망 이벤트를 전달한다.

## 2. 전투 처리 흐름

```txt
공격 입력 또는 AI 공격 실행
→ 공격 가능 상태 확인
→ 공격 모션 / 데우스 발동 연출 실행
→ 공격 판정 생성
→ 명중 대상 확인
→ 피해 계산 요청
→ 피해 적용
→ 효과 또는 상태이상 적용
→ 피격 반응 처리
→ HP 0 이하 여부 확인
→ 사망 / 처치 이벤트 발생
→ 저장, 퀘스트, 레벨, 회차 시스템에 이벤트 전달
```

## 8.3 Unity 전투 실행 데이터 기준

전투 시스템은 모듈 ID, 보스 ID, 퀘스트 ID로 분기하지 않는다. 전투는 실행 요청으로 전달된 Definition을 처리하고 결과 이벤트만 발생시킨다.

| 데이터 | 전투 시스템 책임 | 비고 |
| --- | --- | --- |
| AbilityDefinition | ActionDefinition 실행 순서를 AbilityExecutor에 위임 | 모듈/보스/ENEMY 공통 |
| HitboxDefinition | 판정 형태, 크기, 유지 시간, 중복 명중 제한 처리 | 근접/범위/낙하물 공통 |
| ProjectileDefinition | 투사체 이동, 수명, 충돌 시 Hitbox 생성 | 원거리/소환체 공통 |
| EffectDefinition | 명중 후 적용할 효과 목록 전달 | 실제 지속 관리는 EffectSystem |
| DamageFormulaDefinition | 피해 계산 방식 선택 | 기본값은 고정 피해 또는 공격력 계수 |

```txt
AbilityRequested
→ AbilityExecutor가 ActionDefinition 순차 실행
→ SpawnHitbox / SpawnProjectile / MoveCaster / PlaySfx 등 처리
→ HitConfirmed 이벤트 발생
→ DamageCalculator가 피해 계산
→ EffectSystem이 EffectDefinition 적용
→ EnemyKilled / BossKilled / PlayerDead 이벤트 발생
```

전투 시스템은 UI, 퀘스트, 모듈 트리 해금을 직접 호출하지 않는다. 모든 후속 연결은 이벤트 버스를 통해 처리한다.
