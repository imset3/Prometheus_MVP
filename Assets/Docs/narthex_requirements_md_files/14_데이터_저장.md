# 14. 데이터 저장

## 1. 데이터 저장 정의

데이터 저장은 영구 해금 데이터, 회차 데이터, 설정 데이터를 분리하여 관리하는 기능이다.

로그라이크식 진행 구조이므로 모든 진행 데이터가 영구 저장되는 것은 아니다.

보스 모듈 트리 해금처럼 회차가 바뀌어도 유지되어야 하는 데이터만 영구 저장 데이터로 관리한다.

레벨, 경험치, 모듈 포인트, 모듈 해금/강화/장착, 송신탑 상태, 스테이지 진행, 퀘스트 진행은 회차 데이터로 관리하고 사망 시 초기화한다.

## 2. 저장 데이터 분류

| 분류 | 유지 기준 | 예시 |
| --- | --- | --- |
| 영구 데이터 | 새 게임 초기화 전까지 유지 | 보유 모듈 트리, 보스 처치 해금 기록, 누적 기록 |
| 회차 데이터 | 현재 회차 동안만 유지 | 레벨, 경험치, 모듈 포인트, 스테이지 진행, 송신탑 상태 |
| 설정 데이터 | 게임 진행과 별개로 유지 | 사운드, 화면, 키 설정 |
| 고정 데이터 | 저장하지 않음 | 레벨 테이블, 모듈 기본 데이터, ENEMY 기본 능력치 |

## 7. 저장하지 않는 데이터

| 항목 | 제외 이유 |
| --- | --- |
| 모듈 기본 효과 | 고정 기획 데이터이므로 모듈 데이터에서 참조한다. |
| 레벨 필요 경험치 표 | 고정 밸런스 데이터이다. |
| ENEMY 기본 능력치 | ENEMY 데이터 테이블에서 관리한다. |
| 보스 패턴 상세 | 보스 패턴 문서에서 정의하고 런타임 상태만 회차 데이터로 저장한다. |
| UI 이미지와 사운드 리소스 | 리소스 데이터이며 저장 데이터가 아니다. |

## 7.1 Unity Definition 저장 경계

ScriptableObject Definition Asset은 고정 데이터이며 저장 파일에 직접 기록하지 않는다. 저장 파일에는 stableId와 런타임 값만 기록한다.

| 대상 | 저장하는 값 | 저장하지 않는 값 |
| --- | --- | --- |
| ModuleDefinition | moduleId, unlock 여부, 강화 단계, 장착 슬롯 | ScriptableObject 참조, 에셋 경로 |
| ModuleTreeDefinition | treeId, 접근 권한 여부 | 트리 에셋 전체 |
| EffectDefinition | effectId, 남은 시간, 중첩 수 | 효과 에셋 참조 |
| QuestDefinition | questId, 상태, 조건별 진행도, 보상 지급 여부 | 조건/보상 에셋 전체 |
| BossDefinition | bossId, 현재 회차 처치 여부, 런타임 카운터 | 보스 패턴 에셋 전체 |
| SettingsDefinition | 설정 키, 값 | UI 프리셋 에셋 참조 |

고정 데이터 ID를 변경해야 할 경우 저장 데이터 마이그레이션 규칙을 먼저 작성한다.
