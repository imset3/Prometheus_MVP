# 요구사항 총 내용본

## 확정 설계 기준

- 대상 범위는 튜토리얼과 CHAPTER_01이다.
- 진행 구조는 로그라이크식 회차 진행이다.
- 플레이어 사망 시 회차 데이터는 초기화하고, CHAPTER_01 STG-001부터 다시 시작한다.
- 송신탑은 저장/진행 보조 장치이며 부활 지점으로 사용하지 않는다.
- 이전 성장 시스템은 사용하지 않는다.
- 플레이어는 기본 모듈 트리를 가지고 시작한다.
- 보스 처치 시 보스 모듈 트리를 영구 해금한다.
- 모듈 포인트는 회차 자원이며 사망 시 초기화한다.
- 기본 모듈 트리와 보스 모듈 트리 안의 모듈은 회차 중 모듈 포인트로 해금/강화한다.
- 모듈 사용 흐름은 플레이어 입력, 데우스 발동, 전투 판정 처리 순서로 연결한다.
- 퀘스트, 보스 패턴, 모듈, 효과는 하드 코딩하지 않고 Unity ScriptableObject 기반 Definition Asset으로 정의한다.
- Unity 6.3 URP 기준으로 개발하며, URP Render Graph, Addressables, URP Material/Shader 호환성을 검증 기준에 포함한다.
- 2D 액션 플랫포머 제작 기준은 URP 2D Renderer, Physics2D, StageDefinition, RoomDefinition, HitboxDefinition, PlayerMotorDefinition을 기준으로 통일한다.
- 보스 패턴 상세는 별도 입력 후 작성한다. 현재 문서에서는 최종 보스 패턴 상세를 제외한 연결 구조만 정의한다.
