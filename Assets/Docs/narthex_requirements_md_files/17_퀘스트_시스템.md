# 17. 퀘스트 시스템

## 1. 퀘스트 시스템 정의

퀘스트 시스템은 튜토리얼과 CHAPTER_01에서 플레이어가 수행해야 할 목표를 안내하고, 목표 달성 여부를 판단하여 다음 진행 단계로 연결하는 시스템이다.

퀘스트는 튜토리얼 진행, 스테이지 클리어, ENEMY 처치, 보스 처치, 모듈 사용, 송신탑 활성화, 포탈 이동 같은 진행 조건을 관리한다.

퀘스트 진행 상태는 회차 데이터이며 사망 또는 회차 재시작 시 초기화한다.

보스 처치 퀘스트 완료 시 보스 모듈 트리 해금 요청을 발생시킬 수 있다. 해금된 보스 모듈 트리 접근 권한은 영구 데이터로 저장한다.

## 12.1 Unity 퀘스트 데이터 구현 기준

퀘스트는 `QuestDefinition`이 조건과 보상을 참조하고, QuestManager는 이벤트를 받아 진행도만 갱신하는 구조로 구현한다.

| 구현 요소 | 역할 | 예시 |
| --- | --- | --- |
| QuestDefinition | 퀘스트 ID, 표시명, 시작 조건, 완료 조건, 보상, 다음 퀘스트 정의 | QST-BOSS-001 |
| QuestConditionDefinition | 완료 조건 타입과 대상 ID, 요구 수량 정의 | BossKilled:BOSS-CH01-MID-MINOTAUR |
| QuestRewardDefinition | 지급 보상 타입과 대상 ID, 수량 정의 | BossModuleTreeUnlock:TREE-BOSS-MINOTAUR |
| QuestRuntimeState | 상태, 조건별 진행도, 보상 지급 여부 | InProgress, 1/1 |
| QuestConditionEvaluator | 이벤트를 조건 진행도로 변환 | EnemyKilled 수량 증가 |
| QuestRewardExecutor | 보상을 실제 시스템 요청으로 변환 | ModuleTreeUnlockRequested 발생 |

```txt
GameEventBus 이벤트 수신
→ QuestConditionEvaluator가 관련 QuestConditionDefinition 검색
→ QuestRuntimeState 진행도 갱신
→ 완료 조건 조합 방식 All / Any / Sequence 평가
→ QuestCompleted 이벤트 발생
→ QuestRewardExecutor가 RewardDefinition 실행
→ QuestRewarded 이벤트 발생
```

QuestManager는 UI, 저장, 모듈 트리 시스템을 직접 참조하지 않는다. 보상 처리는 RewardDefinition과 이벤트로 연결한다.
