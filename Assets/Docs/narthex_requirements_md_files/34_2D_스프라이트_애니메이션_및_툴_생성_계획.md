# 34. 2D 스프라이트 애니메이션 및 툴 생성 계획

## 1. 목표와 전제

현재 튜토리얼은 기능 검증용 도형으로 구성되어 있다. 다음 시각화 단계에서는 플레이어, 적, 송신탑을 2D 스프라이트와 애니메이션으로 교체하되, 게임 로직은 기존 Host와 Definition을 그대로 유지한다.

- 씬과 Prefab에는 `SpriteRenderer`, `Animator`, 효과 Anchor를 미리 배치한다.
- 런타임에는 새 GameObject, Component, Animator를 생성하지 않는다.
- 프로젝트에 설치된 `com.unity.2d.animation`, `com.unity.2d.aseprite`, `com.unity.2d.psdimporter`, `com.unity.2d.sprite`를 우선 사용한다.
- 원본 아트가 없을 때는 도형 placeholder를 유지하고, 생성 이미지는 검토 후 교체한다.

## 2. 에셋 규칙

| 대상 | 원본 경로 | 산출 경로 | 필수 상태 |
| --- | --- | --- | --- |
| Player | `Assets/_Project/Art/Characters/Player/Source` | `.../Sprites`, `.../Animations` | Idle, Run, Jump, Fall, Dash, Attack, Hit, Dead |
| Enemy | `Assets/_Project/Art/Characters/Enemy/Source` | `.../Sprites`, `.../Animations` | Idle, Move, Attack, Hit, Dead |
| Relay | `Assets/_Project/Art/Props/Relay/Source` | `.../Sprites`, `.../Animations` | EnemyIdle, Activate, PlayerIdle |
| Effects | `Assets/_Project/Art/VFX/Source` | `.../Sprites`, `.../Animations` | Hit, DashTrail, RelayPulse |

원본 시트 파일명에는 격자 크기를 `열x행`으로 붙인다. 예: `PLAYER_RUN_4x4.png`는 4열 x 4행, 총 16프레임이며 `RELAY_ACTIVATE_6x10.png`는 6열 x 10행, 총 60프레임이다. 자동 인식이 안 되는 파일은 툴에서 열과 행을 직접 입력한다.

생성 프레임은 `Assets/_Project/Art/Generated/<원본명>/Frames`에 `원본명_000.png` 형식으로 저장한다. 원본 PNG는 변경하지 않는다.

## 3. 생성 툴 계획

### A. Sprite Import Profile Tool

Editor 메뉴 `Narthex/Art/Apply Sprite Import Profile`을 만든다.

- Texture Type = Sprite (2D and UI)
- Sprite Mode = Multiple
- Filter Mode = Point 또는 프로젝트 선택값
- Compression = None (MVP), Pixel Per Unit = 캐릭터/소품 프로필값
- Pivot, Slice 규칙, Import Profile 결과를 검사한다.

입력은 선택한 PNG, PSD, Aseprite 에셋이며 출력은 Importer 설정뿐이다. 원본 픽셀을 변경하지 않는다.

### B. Sprite Sheet Animation Builder (확정)

Editor 메뉴 `Narthex/Art/Sprite Sheet Animation Builder`를 만든다. 한 장에 이어진 스프라이트 시트를 넣으면 분리, 배경 제거, Clip 생성을 한 번에 수행한다.

- 파일명의 `4x4`, `6x10` 같은 `열x행` 표기를 자동 인식하고, 필요하면 수동 열/행 값을 사용한다.
- 시트를 위에서 아래, 왼쪽에서 오른쪽 순서로 프레임 PNG로 자른다.
- 각 프레임의 좌상단 색을 기준으로, 외곽에서 이어진 유사 색상만 flood fill로 투명 처리한다. 따라서 균일한 흰색/초록색/단색 배경은 지우되 캐릭터 내부의 빈 공간이나 분리된 유사 색상은 보존한다.
- 복잡한 그라데이션 배경, 그림자가 배경과 이어진 원본은 자동 제거 대상이 아니다. 이 경우 투명 원본을 사용하거나 허용 오차를 조정한다.
- 분리한 Sprite 배열을 프레임명 순서로 정렬하고, 상태별 `.anim`을 `Animations` 폴더에 생성 또는 갱신한다.
- FPS와 Loop 값을 창에서 설정한다. 기본값은 Idle 8, Run 12, Dash 18, Attack 14, Hit 10, Relay Activate 12이다.
- 생성 후 첫 프레임, 마지막 프레임, SpriteRenderer 바인딩, 0 FPS 여부를 검사한다.

### C. Animator Controller Builder

Editor 메뉴 `Narthex/Art/Build Animator Controller`를 만든다.

- 캐릭터 파라미터: `MoveSpeed`, `VerticalSpeed`, `Grounded`, `Dash`, `Attack`, `Hit`, `Dead`.
- 송신탑 파라미터: `Activated`.
- Any State 전환은 Hit와 Dead만 허용하고, 공격과 대시는 종료 조건을 가진다.
- Controller는 기존 Prefab/씬의 사전 배치 Animator에만 연결한다.

### D. Animation Validation Tool

Editor 메뉴 `Narthex/Validation/Validate Sprite Animation Setup`을 만든다.

- 필수 상태별 Clip/Controller/Renderer/Animator 참조를 검사한다.
- 빈 Clip, 중복 프레임, 누락된 Sprite, 잘못된 Loop, 0 FPS를 오류 처리한다.
- Runtime 코드의 `new GameObject`, `Instantiate`, `AddComponent` 금지 검사도 함께 수행한다.

## 4. 적용 순서

1. Player Idle/Run/Jump/Dash/Attack 6개 상태를 만든다.
2. PlayerRoot의 도형 Visual child를 PlayerSpriteRoot Prefab으로 교체한다.
3. TutorialEnemy Idle/Attack/Hit/Dead를 만든다.
4. TutorialRelay의 EnemyIdle/Activate/PlayerIdle을 만든다.
5. DashTrail, Hit, RelayPulse를 Sprite 기반 VFX child로 사전 배치한다.
6. 각 단계마다 Play Mode 화면과 Validator 결과를 캡처한다.

### 4.1 현재 적용된 프로토타입 모션

- 원본 스프라이트가 들어오기 전에도 전투 읽기성을 확보하기 위해 `CombatVisualMotionHost`가 사전 배치된 Visual Transform을 직접 움직인다.
- 플레이어의 `Enter` 공격과 모든 전투 대상의 `HitConfirmed` 이벤트는 짧은 전진, 회전, squash/stretch, 베기 효과 표시를 발생시킨다.
- 피격 시 흔들림과 압축, 사망 시 쓰러짐 모션을 제공한다. 헬테는 `HelteBossPatternHost.PatternStarted`에 연결돼 패턴 시작에도 모션을 재생한다.
- 이 프로토타입은 Animator/SpriteRenderer 교체 전의 임시 시각 피드백이며, 런타임 GameObject/Component 생성 없이 씬에 미리 배치한 Visual과 Effect만 사용한다.

### 4.2 튜토리얼 Definition 참조 기준

- 실제 튜토리얼 실행용 ScriptableObject는 `Assets/_Project/GameData/Tutorial/RuntimeDefinitionsV2` 아래의 Unity Editor 생성 에셋을 사용한다.
- 이 경로에는 능력 1개, 모듈 1개, 모듈 트리 2개, 조건 8개, 퀘스트 8개가 포함된다.
- `TutorialModuleUseHost`, `ModuleTreeManagerHost`, `TutorialQuestSequenceHost`, `GameContentRegistry`는 모두 위 에셋을 Inspector 참조로 사용한다.
- 사람이 직접 작성한 YAML에서 `m_Script: {fileID: 0}`인 Definition 에셋은 런타임 참조로 사용하지 않는다. 신규 Definition은 Unity Editor의 `ScriptableObject.CreateInstance`와 `AssetDatabase.CreateAsset` 또는 검증된 에디터 툴로 생성한다.

## 5. 생성 이미지 사용 기준

생성 이미지는 컨셉 확인용 PNG 또는 스프라이트 시트로만 사용한다. 최종 캐릭터 애니메이션은 프레임 일관성, 피벗, 실루엣, 타격 타이밍을 검수한 뒤 채택한다. 코드나 런타임 생성으로 프레임을 조립하지 않는다.

## 6. 완료 기준

- Player, Enemy, Relay의 필수 상태가 각각 Clip과 Controller로 존재한다.
- 모든 Animator와 SpriteRenderer는 Prefab 또는 씬 계층에 사전 배치돼 있다.
- 게임플레이 Host는 Animator 파라미터만 변경하며 프레임/오브젝트를 생성하지 않는다.
- Sprite Animation Validator와 핵심 PlayMode 검증이 통과한다.
