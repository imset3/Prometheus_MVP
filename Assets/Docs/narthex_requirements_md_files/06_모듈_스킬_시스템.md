# 06. 모듈(스킬) 시스템

## 1. 모듈(스킬) 시스템 정의

모듈(스킬)은 플레이어가 회차 중 모듈 트리에서 해금, 강화, 장착하여 사용하는 기능 단위이다.

일반 공격과 방향 공격은 모듈(스킬)이 아니다.

모듈은 기본 모듈 트리 또는 보스 모듈 트리에 포함된다.

플레이어는 입력을 통해 모듈 사용을 요청하고, 데우스(펫)는 모듈 발동 위치와 연출을 담당한다. 전투 시스템은 공격 판정, 명중, 피해, 효과 적용을 처리한다.

## 2. 모듈 발동 책임 분리

| 책임 | 담당 | 설명 | 우선순위 |
| --- | --- | --- | --- |
| 입력 요청 | 플레이어 / 입력 시스템 | C/V 입력으로 장착 모듈 사용 요청을 발생시킨다. | 상 |
| 발동 주체 연출 | 데우스(펫) | 데우스 정면 또는 지정 위치에서 모듈 발동 연출을 출력한다. | 상 |
| 비용/쿨타임 확인 | 모듈 시스템 | 장착 여부, 쿨타임, 사용 가능 상태를 확인한다. | 상 |
| 판정 생성 | 전투 시스템 | 모듈 종류에 맞는 공격/이동/효과 판정을 생성한다. | 상 |
| 피해/효과 처리 | 전투 시스템 / 효과 분류 | 명중 대상에게 피해, 상태이상, 피격 반응을 적용한다. | 상 |

## 9.1 Unity 데이터 주도 구현 기준

모듈은 코드에 직접 효과와 판정을 작성하지 않고 `ModuleDefinition`이 실행할 `AbilityDefinition`과 적용할 `EffectDefinition`을 참조하는 구조로 구현한다.

| 구현 요소 | 역할 | 연결 기준 |
| --- | --- | --- |
| ModuleDefinition | 모듈 ID, 이름, 소속 트리, 슬롯, 해금 비용, 강화 단계 정의 | 모듈 트리와 UI가 참조 |
| AbilityDefinition | 직접 사용형 모듈이 실행하는 행동 묶음 | ModuleDefinition이 참조 |
| EffectDefinition | 패시브, 강화, 상태이상, 피해 후속 효과 | ModuleDefinition 또는 AbilityDefinition이 참조 |
| UpgradeModifierDefinition | 강화 단계별 수치/판정/쿨타임 변경 | ModuleDefinition 내부에서 단계별 참조 |
| ModuleRuntimeState | 현재 회차의 해금/장착/강화 상태 | 저장 시스템이 ID와 단계만 저장 |

```txt
플레이어 C/V 입력
→ 장착 슬롯의 moduleId 조회
→ ModuleDefinition 로드
→ 쿨타임 / 해금 / 행동 가능 상태 확인
→ AbilityDefinition 실행 요청
→ AbilityExecutor가 ActionDefinition 실행
→ 전투 시스템이 Hitbox / Projectile / Effect 처리
```

모듈 시스템은 특정 모듈 ID로 분기하지 않는다. 새 모듈은 Definition Asset 추가와 모듈 트리 연결만으로 추가되어야 한다.
