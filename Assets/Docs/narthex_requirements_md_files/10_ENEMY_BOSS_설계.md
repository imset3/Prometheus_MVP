# 10. ENEMY / BOSS 설계

## 1. 설계 범위

이 문서는 튜토리얼과 CHAPTER_01에 등장하는 ENEMY/BOSS의 구조, 보스 패턴, 처치 보상 연결을 정의한다.

헬테와 CHAPTER_01 중간 보스는 패턴을 확정한다.

CHAPTER_01 최종 보스는 입력 대기 상태로 두며, 나중에 패턴 내용만 추가할 수 있도록 슬롯을 유지한다.

## 5.1 Unity 보스 데이터 구현 기준

보스 패턴은 코드에 직접 작성하지 않고 `BossDefinition`, `BossPatternDefinition`, `AbilityDefinition` 조합으로 구현한다.

| Definition | 역할 | 런타임 처리 |
| --- | --- | --- |
| BossDefinition | 보스 ID, 표시명, 페이즈, 보상 트리, 패턴 목록 정의 | BossDirector가 로드 |
| BossPhaseDefinition | 페이즈 진입 조건과 사용 가능한 패턴 목록 정의 | HP, 시간, 이벤트 조건으로 전환 |
| BossPatternDefinition | Pattern ID, 발동 조건, 쿨타임, 우선순위, 후속 규칙 정의 | 조건 평가 후 선택 |
| AbilityDefinition | 실제 이동, 판정, 투사체, 소환, 사운드, VFX 실행 묶음 | AbilityExecutor가 실행 |
| PatternRuntimeState | 쿨타임, 사용 횟수, 카운터, 현재 실행 상태 | 회차 중 런타임 상태로만 유지 |

```txt
BossDirector
→ 현재 Phase 확인
→ PatternDefinition 조건/쿨타임/우선순위 평가
→ 선택된 Pattern의 AbilityDefinition 실행
→ CombatSystem이 판정/피해/효과 처리
→ PatternRuntimeState 갱신
```

BossDirector는 `BOSS-TUTO-HELTE`, `BOSS-CH01-MID-MINOTAUR` 같은 ID별 분기를 가지지 않는다. 새로운 보스나 최종 보스는 Definition Asset 추가와 연결로 확장한다.
