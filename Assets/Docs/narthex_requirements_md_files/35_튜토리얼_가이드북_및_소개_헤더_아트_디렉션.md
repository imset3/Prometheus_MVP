# 프로메테우스 튜토리얼 가이드북

## 1. 게임 한 줄 소개

`프로메테우스`는 스팀펑크 세계에서 금지된 기술을 탈취해 지배자에게 맞서는 2D 액션 플랫포머다. 플레이어는 이동과 전투를 익히고, 모듈을 장착해 전투 방식을 만들며, 첫 보스 헬테를 처치해 보스 모듈 트리를 영구 해금한다.

## 2. 튜토리얼 목표

튜토리얼의 최종 목표는 헬테를 처치하는 것이다. 그 과정에서 이동, 점프, 공격, 대시, 모듈, 모듈 트리, 송신탑을 순서대로 익힌다. 보스 처치 후에는 `TREE-BOSS-HELTE`가 영구 데이터에 기록되고 다음 목적지 `CHAPTER_01`이 열린다.

## 3. 현재 빌드 조작법

| 행동 | 키 | 설명 |
| --- | --- | --- |
| 이동 | `A` / `D`, 방향키 좌우 | 좌우로 이동한다. |
| 점프 | `Space` | 발판을 넘거나 공격을 피한다. |
| 대시 | `Left Shift` | 짧게 거리를 벌리거나 위험 구간을 통과한다. |
| 일반 공격 | `Enter` | 전방 근접 공격을 사용한다. |
| 튜토리얼 모듈 | `2` | 장착된 튜토리얼 모듈을 사용한다. |
| 상호작용 | `F` | 송신탑 등 가까운 대상과 상호작용한다. |
| 모듈 트리 | `I` | 기본 모듈 트리 패널을 열거나 닫는다. |

`X`, `C`, `V` 등은 기획서의 확장 입력 후보이며, 위 표는 현재 Unity 빌드에 적용된 입력을 기준으로 한다.

입력이 전혀 반응하지 않으면 Unity Inspector의 `PlayerRoot > PlayerInput > Notification Behavior`가 `Invoke C# Events`인지 먼저 확인한다. 이 빌드는 해당 설정을 요구하며, 시작 시에도 잘못된 설정을 자동 보정한다.

## 4. 처음부터 끝까지 따라가기

### Step 1. 이동하기

화면의 현재 목표가 `Move with A / D`로 표시되면 `A` 또는 `D`를 눌러 이동한다. 이 단계는 캐릭터의 기본 이동 감각과 카메라 추적을 확인하는 구간이다.

### Step 2. 점프하기

현재 목표가 `Jump with Space`로 바뀌면 `Space`를 눌러 점프한다. 낮은 장애물과 단차는 대시보다 점프로 먼저 넘는 것이 안전하다.

### Step 3. 일반 공격하기

`Attack with Enter`가 표시되면 `Enter`로 전방을 공격한다. 적과 겹치기 직전보다 한 걸음 떨어진 거리에서 공격하면 적의 공격 판정과 겹칠 위험이 줄어든다.

### Step 4. 대시하기

`Dash with Shift`가 표시되면 `Left Shift`를 눌러 대시한다. 대시는 이동 속도를 높이는 동시에 위험한 공격 범위를 빠져나오는 기본 회피 수단이다.

### Step 5. 모듈 사용하기

`Use module with 2`가 표시되면 `2`를 눌러 튜토리얼용 모듈을 사용한다. 모듈은 일반 공격과 별개의 능력 슬롯이며, 이후 회차에서는 해금과 장착 구성이 전투 전략을 만든다.

### Step 6. 모듈 트리 열기

`Open module tree with I`가 표시되면 `I`를 누른다. 패널이 열리면 기본 트리에 접근한 상태다. 현재 튜토리얼에는 선택한 모듈을 해금하는 버튼이 있으며, 모듈 포인트와 선행 조건이 충족될 때만 해금된다.

### Step 7. 송신탑 활성화하기

`Activate relay with F` 목표가 표시되면 송신탑의 범위 안으로 들어가 `F`를 누른다. 적이 점유하던 송신탑이 플레이어 상태로 바뀌면 다음 구역과 보스전이 열린다. 그 이전에 송신탑에 도착해도 점령은 진행되지 않는다.

### Step 8. 헬테 처치하기

송신탑을 활성화하면 헬테가 나타나고 목표가 `Defeat Helte`로 바뀐다. 헬테는 체력 절반을 기준으로 패턴 구성이 바뀐다.

| 구간 | 패턴 | 대응 |
| --- | --- | --- |
| 1페이즈 | 근접 2연타 | 공격 범위 밖으로 한 걸음 물러나거나 대시로 이탈한다. |
| 1페이즈 | 블링크 후 X자 베기 | 블링크 직후 바로 접근하지 말고, 약간의 예고 뒤 나오는 베기 범위를 피해 움직인다. 블링크 이동 자체에는 피해가 없다. |
| 2페이즈 | 칼 3개 소환 후 블링크 | 칼은 발사 시점의 플레이어 위치를 노리므로 한 자리에 머물지 말고 좌우로 이동한다. 이후 블링크 베기를 대비해 거리를 유지한다. |

일반 공격을 두 번 사용한 뒤 특수 패턴이 이어진다. 무리해서 연속 공격하기보다 특수 패턴 뒤의 빈틈에 `Enter` 공격과 모듈을 함께 사용하는 방식이 안정적이다. 공중에서 `Space`를 누르고 있으면 낙하 속도를 줄이는 활공이 적용되어 발판과 적의 공격을 피할 시간을 만들 수 있다.

## 5. 사망과 진행 저장

튜토리얼 중 플레이어가 쓰러지면 시작 지점으로 돌아간다. 송신탑 점령 퀘스트를 완료한 뒤에는 송신탑 뒤의 체크포인트에서 재시작한다. 완료한 퀘스트와 점령한 송신탑, 이미 등장한 헬테는 유지하며 플레이어와 전투 대상의 HP 및 전투 상태만 초기화한다. 게임을 다시 실행해도 첫 미완료 퀘스트부터 이어지며, 송신탑 ID만 남은 저장 도중 종료 상황은 안전하게 점령 전 상태로 시작한다. 헬테 처치를 완료하면 튜토리얼 완료 상태, 헬테 처치 기록, `TREE-BOSS-HELTE` 해금은 영구 저장된다.

## 6. 전시용 30초 안내 대본

> A와 D로 이동하고 Space로 점프합니다. Enter는 기본 공격, Shift는 대시, 2는 장착한 모듈입니다. I로 모듈 트리를 열고, 송신탑 가까이에서 F를 눌러 활성화하세요. 마지막으로 헬테의 블링크와 칼 소환을 피하면서 처치하면 보스 모듈 트리가 해금됩니다.

## 7. 소개 헤더 이미지 아트 디렉션

### 추천 콘셉트: "탈취한 빛으로 관문을 연다"

가로형 헤더 한 장에서 주인공, 금지 기술, 헬테, 산업 도시의 압박감을 동시에 보여준다. 화면 왼쪽 전경에는 짧은 창 또는 총검을 든 플레이어가 낮은 자세로 서고, 오른쪽 중경에는 블링크 직후 잔상을 남긴 헬테를 배치한다. 두 인물 사이에는 청록색 에너지 코어와 얇은 금빛 기계 회로가 대각선으로 이어져, "기술을 빼앗아 맞선다"는 게임의 핵심을 읽히게 한다.

| 항목 | 권장안 |
| --- | --- |
| 캔버스 | `1920 x 720` 또는 Steam 캡슐용 `2310 x 870` |
| 구도 | 왼쪽 플레이어, 오른쪽 헬테, 중앙은 에너지 충돌과 로고 안전 영역 |
| 배경 | 철골 아치, 증기 배관, 어두운 공장 도시, 멀리 보이는 붉은 경보등 |
| 주조색 | 차가운 청록/강철 회색을 바탕으로 금빛 회로와 적색 경보등을 포인트로 사용 |
| 광원 | 아래쪽 청록 에너지와 오른쪽 붉은 역광의 충돌. 인물 실루엣은 명확히 유지 |
| 타이포 영역 | 중앙 상단 35%를 비교적 비워 `NARTHEX` 로고와 짧은 문구를 올릴 수 있게 유지 |

### 만들 때 지킬 것

- 실제 플레이어와 헬테가 충분히 크게 보여야 한다. 분위기만 있는 배경 이미지는 피한다.
- 글자는 이미지 생성 단계에서 넣지 않는다. 로고와 문구는 후편집으로 올려야 오탈자와 재배치가 쉽다.
- 가장 밝은 청록 에너지는 중앙 로고 영역 전체를 덮지 않고, 인물 사이를 잇는 선으로 제한한다.
- 보스의 얼굴 또는 핵심 실루엣은 오른쪽 가장자리에서 잘리지 않게 한다.
- 모바일 크롭에서는 중앙 1080px 안에 두 인물의 상반신과 에너지 코어가 남도록 설계한다.
