# 02. 입력 시스템

## 1. 입력 분류 조건

입력은 플레이어가 누르는 키 또는 버튼과 그로 인해 발생하는 요청 이벤트만 관리한다.

입력 문서에서는 피해, 판정, 이동 거리, 모듈 효과, 저장 처리를 정의하지 않는다.

키 변경은 설정 시스템에서 관리하며, 입력 시스템은 현재 적용된 키 바인딩을 기준으로 요청 이벤트를 발생시킨다.

## 2. 입력 키

| 기능 | 기본 키 | 보조 입력 | 요청 이벤트 |
| --- | --- | --- | --- |
| 좌우 이동 | A / D | 왼쪽 / 오른쪽 방향키 | MoveHorizontal |
| 위 입력 | W | 위 방향키 | AimUp / LookUp |
| 아래 입력 | S | 아래 방향키 | AimDown / LookDown |
| 점프 / 더블 점프 | Space | - | Jump |
| 활공 | Space 길게 누름 | - | GlideHold |
| 대시 | Shift | - | Dash |
| 일반 공격 | X | Left Mouse | BasicAttack |
| 모듈(스킬) 1 | C | - | UseModuleSlot1 |
| 모듈(스킬) 2 | V | - | UseModuleSlot2 |
| 상호작용 | F | - | Interact |
| 지도 | Tab | - | ToggleMap |
| 모듈 트리 / 인벤토리 | I | - | OpenModuleTree |
| 일시정지 | Esc | - | Pause |

## 3. 입력 요구사항

| ID | 기능 | 요구사항 | 연결 시스템 |
| --- | --- | --- | --- |
| INP-01 | 이동 입력 | 좌우 이동 요청을 발생시킬 수 있어야 한다. | 이동 |
| INP-02 | 점프 입력 | 점프/더블 점프 요청을 발생시킬 수 있어야 한다. | 이동 |
| INP-03 | 활공 입력 | Space 유지로 활공 요청을 발생시킬 수 있어야 한다. | 이동 |
| INP-04 | 대시 입력 | 대시 요청을 발생시킬 수 있어야 한다. | 이동 / 전투 |
| INP-05 | 일반 공격 | 일반 공격 요청을 발생시킬 수 있어야 한다. | 전투 / 무기 |
| INP-06 | 모듈 사용 | 장착된 모듈 슬롯 사용 요청을 발생시킬 수 있어야 한다. | 모듈 / 전투 |
| INP-07 | 상호작용 | 송신탑, 포탈, UI 상호작용 요청을 발생시킬 수 있어야 한다. | 송신탑 / 포탈 / 퀘스트 |
| INP-08 | 지도 | 지도 UI 열기/닫기 요청을 발생시킬 수 있어야 한다. | 지도 / UI |
| INP-09 | 모듈 트리 | 모듈 트리 UI 열기 요청을 발생시킬 수 있어야 한다. | 모듈 트리 / UI |
| INP-10 | 일시정지 | 일시정지 메뉴 요청을 발생시킬 수 있어야 한다. | UI / 설정 |
