# 08. 효과 분류

## 1. 효과 분류 정의

효과는 전투 중 대상에게 발생하는 상태 변화 또는 반응이다.

입력 키, UI 화면, 모듈 트리 해금, 모듈 해금/강화 자체는 효과로 분류하지 않는다.

## 2. 효과 대분류

| 효과 분류 | 설명 | 예시 | 우선순위 |
| --- | --- | --- | --- |
| 행동 상태이상 | 대상의 행동, 이동, 자세, 피격 반응을 직접 방해하거나 변화시키는 효과 | 넉백, 띄우기, 경직, 행동 제한 | 상 |
| 전투 상태이상 | 대상에게 일정 시간 동안 전투 수치 또는 피해 효과를 적용하는 효과 | 감전, 화상, 빙결, 둔화, 지속 피해, 중독 | 상 |

## 5.1 Unity EffectDefinition 구현 기준

효과는 모듈, 보스 패턴, ENEMY 공격, 보상, 버프가 공통으로 참조하는 독립 데이터로 구현한다.

| 데이터 항목 | 설명 | 예시 |
| --- | --- | --- |
| effectId | 효과 고유 ID | EFF-KNOCKBACK-001 |
| effectType | Damage / Knockback / Stun / StatModifier / DamageOverTime 등 | Knockback |
| targetFilter | 적용 대상 조건 | PlayerOnly / EnemyOnly / BossOnly |
| duration | 지속 시간 | 0.3초 |
| tickInterval | 반복 적용 간격 | 1초 |
| stackRule | 중첩 방식 | RefreshDuration / StackCount / Ignore |
| resistRule | 보스/정예 대상 저항 규칙 | BossActionResist |
| statModifiers | 스테이터스 변경 목록 | 이동 속도 -30% |
| actionOnApply | 적용 즉시 실행할 행동 | 넉백 적용 |
| actionOnExpire | 종료 시 실행할 행동 | 이동 제한 해제 |

효과 적용 순서는 `HitConfirmed → EffectDefinition 조회 → 저항/면역 검사 → 중첩 규칙 처리 → RuntimeEffectInstance 생성 또는 갱신`으로 통일한다.
